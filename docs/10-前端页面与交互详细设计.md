# 前端页面与交互详细设计

## 前端目标

前端要做到两个极端同时成立：

1. 聊天主界面极简，像聊天软件，不像后台。
2. 管理页面完整、清晰、可维护，能管理成员、组织、资产、任务和系统。

所有页面都必须服从已确定硬规则：

```text
聊天页不显示组织和壳
聊天页只显示当前聊天对象的人名、头像、状态
管理页完整存在
系统管理固定
资产二级分类固定
公司壳首发，但 UI 通过 shell runtime 获取名称
```

```text
┌─────────────────────────────────────────────┐

├───────────────┬─────────────────────────────┤
│ Sidebar       │ Page Content                │
│               │                             │

```

侧边栏可以显示壳映射后的菜单名，但聊天页面顶部不能显示壳和组织。

侧边栏规则：

| 项 | 规则 |
|---|---|
| 菜单 key | 使用固定 key |
| 菜单 label | 从 Shell Runtime 获取 |
| 系统管理 | label 固定“系统管理” |
| 资产管理 | 一级 label 可随壳变化 |
| 资产二级 | 固定大脑、账号、钱包、硬件、知识库 |

## 设计风格

整体风格应该是安静、清晰、现代、低干扰。

```text
不要做营销页
不要做大 hero
不要用过多装饰
不要把主页面做成卡片堆
不要让按钮文字溢出
不要在管理页使用夸张拟人文案
```

推荐：

```text
列表 + 详情
表格 + 筛选
简洁表单
明确状态
轻量确认
可折叠高级设置
```

## 聊天页

### 页面目标

聊天页是用户每天打开最多的页面。它必须让用户感觉“我在和这个人说话”，不是“我在操作组织系统”。

### 结构

```text
ChatPage
  ChatSidebar
    ConversationList
    MemberShortcutList
  ChatMain
    ChatHeader
    MessageList
    PendingApprovalBar
    ChatInput
```

### ChatHeader

只允许：

```text
头像
人名
状态
```

状态枚举：

```text
在线
思考中
执行中
等待确认
离线
```

禁止：

```text
组织名称
公司壳
部门
岗位
权限等级
资产数量
```

### MessageList

支持内容：

```text
普通文本
Markdown 段落
有序/无序列表
表格
代码块
引用摘要
轻量状态行
确认按钮
```

不做：

```text
复杂卡片墙
右侧任务面板
多列仪表盘
过度拟人角色剧场
```

### ChatInput

基础能力：

```text
多行输入
Enter 发送
Shift+Enter 换行
附件入口
发送按钮
停止生成按钮
```

后续能力：

```text
语音输入
拖拽文件
命令建议
成员 @ 提及
```

### ApprovalBar
当任务需要确认时，也是以聊天文字的形式来交互


## 聊天页状态机

```text
idle
typing
submitting
streaming
tool_running
approval_required
paused
completed
failed
```

状态变化：

```text
idle -> submitting -> streaming -> completed
streaming -> tool_running -> streaming
tool_running -> approval_required -> streaming
approval_required -> paused
streaming -> failed
```

UI 规则：

| 状态 | UI |
|---|---|
| submitting | 输入框禁用发送，显示发送中 |
| streaming | 显示停止按钮 |
| tool_running | 顶部状态“执行中”，消息中显示轻量进度 |
| approval_required | 显示确认条，任务暂停 |
| failed | 显示失败原因和重试按钮 |

## Onboarding 页面

首次启动流程：

```text
WelcomePage
  -> CreateOrganizationStep
  -> SelectBrainStep
  -> CreateFirstMemberStep
  -> EnterChat
```

### WelcomePage

只做简单说明，不做营销页。

文案方向：

```text
创建你的个人 AI 组织，先从一个默认助手开始。
```

### CreateOrganizationStep

字段：

```text
组织名称，默认：公司
老板称呼，默认：老板
壳，默认：company，不展示复杂选择
```

### SelectBrainStep

选项：

```text
本地模型
云模型
稍后配置
```

如果用户选择稍后配置，系统可使用占位配置，但聊天页要提示需要配置模型。

### CreateFirstMemberStep

字段：

```text
名字，默认：小曜
头像，默认生成或内置
人格，默认：可靠温暖
大脑，使用上一步选择
```

不要让用户配置权限、资产、Skill。

## 前端 API 类型

所有 API 类型从后端 OpenAPI 或 `packages/core-types` 生成，不手写重复结构。

原则：

```text
服务端 schema 是源头
前端不得私自扩展核心字段
壳 label 只用于展示，不写回业务字段
```

## 验收清单

```text
聊天页顶部只显示人名头像状态
切换公司壳/其他壳后聊天页仍不显示壳
系统管理菜单固定
资产二级分类固定
创建成员首屏只有四类核心字段
账号和钱包 secret 不回显
任务详情可查看 trace
高风险确认能通过键盘操作
所有页面空状态都有下一步动作
```
