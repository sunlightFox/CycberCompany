# 产品设计

## 产品愿景

这款产品要做成用户电脑里的“个人智能体操作系统”。它既能像员工一样执行任务，也能像朋友一样持续聊天，还能像管家一样管理资产、知识、设备和待办。

用户不应该感觉自己在操作一个复杂后台，而应该感觉自己拥有了一组长期存在、会记住自己、会成长、会调用工具做事的 AI 生命体。

## 用户画像

| 用户 | 目标 | 典型需求 |
|---|---|---|
| 个人效率用户 | 把琐碎任务交给 AI | 整理文件、规划日程、生成清单、复盘任务 |
| 知识工作者 | 长期记住项目上下文 | 写方案、做研究、整理资料、持续推进项目 |
| 内容创作者 | 让 AI 参与运营 | 写短视频脚本、标题、发文草稿、账号内容计划 |
| 轻开发用户 | 自动化本地工作 | 运行脚本、读日志、部署服务、生成代码 |
| 家庭用户 | 统一管理生活资源 | 智能家居、提醒、陪伴、生活建议 |
| 陪伴型用户 | 有人味的聊天体验 | 幽默、温暖、可靠、有连续关系感 |

## 核心价值

| 价值 | 产品表现 |
|---|---|
| 会聊天 | 回复自然、结构清晰、情绪匹配、有个性 |
| 会记忆 | 能跨会话记住偏好、项目、关系和历史任务 |
| 会做事 | 能调用文件、浏览器、终端、设备、账号、知识库 |
| 会成长 | 把重复任务沉淀成 Skill，把成功经验写入记忆 |
| 可控 | 高风险动作确认，任务过程可回放，记忆可查看和纠错 |
| 好上手 | 首次使用几分钟内开始聊天，不强迫用户配置复杂权限 |

|---|---|---|
| 聊天页 | 日常主入口 | 单聊、群聊入口、流式回复、确认按钮、任务状态、历史会话 |
| 成员管理 | 管理 AI 生命体 | 创建、编辑、启停、绑定大脑、人格、技能、查看记忆和工作记录 |

## 聊天页设计

### 聊天页目标

聊天页是关系入口，不是后台面板。它必须像一个干净聊天软件，而不是项目管理工具。

### 聊天页元素

| 区域 | 元素 | 规则 |
|---|---|---|
| 顶部 | 头像、人名、状态 | 只展示当前聊天对象，不展示组织和壳 |
| 左侧 | 会话列表 | 可折叠，显示最近会话和成员 |
| 中间 | 消息流 | 文本为主，支持表格、代码块、列表、重点高亮 |
| 底部 | 输入框 | 支持文本、附件、语音后续可加 |
| 消息内 | 轻量按钮 | 仅用于确认、继续、暂停、查看过程等必要动作 |

### 聊天页禁止项

```text
禁止显示组织树
禁止显示壳名称
禁止显示部门管理面板
禁止显示资产卡片墙
禁止在右侧常驻复杂属性面板
禁止把 Skill/MCP 配置塞进聊天页
禁止用大量卡片替代自然回复
```

### 消息表现规范

回复必须根据场景选择结构：

| 场景 | 推荐结构 |
|---|---|
| 普通问答 | 结论先行 + 简短解释 |
| 方案设计 | 标题 + 分层小节 + 表格 + 决策建议 |
| 执行任务 | 我将做什么 + 当前进度 + 结果 + 下一步 |
| 高风险确认 | 动作、影响范围、可撤销性、确认按钮 |
| 情绪陪伴 | 先承接情绪，再给温和建议 |
| 代码/开发 | 文件路径、改动点、测试结果、残留风险 |

聊天质量由 Response Composer 保证，不直接把模型原始输出扔给用户。

## 首次启动流程

目标：五分钟内让用户体验到“它能聊、能记、能做事”。

流程：

1. 打开应用。
2. 显示极简欢迎页。
3. 默认选择公司壳。
4. 输入组织名称，默认“一人公司”。
5. 选择大脑：
   - 本地模型
   - 云模型
   - 混合模式
6. 选择默认主成员：
   - 默认“小曜，首席助理”
   - 可改名、头像、语气
7. 进入聊天页。
8. 系统给出一句自然开场，不展示复杂教程。

欢迎语示例：

```text
老板，我在。你可以直接把要做的事丢给我，也可以先和我聊聊今天想推进什么。
```

## 创建成员流程

创建成员要简化。首屏只展示：

```text
名字
头像
大脑
人格模板
```

可选高级设置折叠：

```text
所属组织
所属部门
角色/岗位
默认技能
可用资产
记忆策略
审批策略
工作风格
模型路由
```

### 默认成员模板

公司壳首发建议内置 5 个模板，但用户不一定首次全启用。

| 名称 | 部门 | 岗位 | 定位 | 默认能力 |
|---|---|---|---|---|
| 小曜 | 总裁办 | 首席助理 | 可靠、温暖、会统筹 | 任务拆解、复盘、协调 |
| 阿珩 | 研发部 | 技术架构师 | 专业、直接、技术强 | 架构、代码、部署、排错 |
| 宁宁 | 产品部 | 产品经理 | 结构清晰、体验敏感 | PRD、需求、路线图 |
| 墨白 | 运营部 | 内容运营 | 创意强、懂传播 | 文案、标题、内容日历 |
| 小栖 | 生活管家部 | 私人管家 | 温柔、细心、陪伴 | 日程、提醒、家居、情绪支持 |

### 创建成员验收

| 验收项 | 标准 |
|---|---|
| 创建成本 | 默认流程 30 秒内完成 |
| 必填项 | 只有名字、大脑、人格模板 |
| 可用性 | 创建后可立即聊天 |
| 安全 | 默认可做低风险动作，高风险仍确认 |
| 记忆 | 自动创建独立成员记忆空间 |

## 成员管理

成员管理是 AI 生命体管理，不只是账号列表。

功能：

```text
创建成员
编辑成员
克隆成员
启用/停用成员
删除或归档成员
分配部门
分配岗位/角色
绑定大脑
绑定人格模板
绑定默认 Skill
查看可用资产
查看记忆
查看工作记录
查看成长记录
调整回复风格
调整审批策略
```

成员详情页分区：

| 分区 | 内容 |
|---|---|
| 基础信息 | 名字、头像、状态、创建时间 |
| 身份信息 | 所属组织、部门、岗位/角色 |
| 大脑 | 模型、路由、成本策略、隐私策略 |
| 人格 | 人格模板、语气、温度、幽默、拟人程度 |
| 记忆 | 私有记忆、可见记忆、写入策略、纠错 |
| 能力 | 默认 Skill、工具、MCP、可用资产 |
| 工作记录 | 最近任务、成功率、失败原因 |
| 安全 | 权限、审批策略、风险限制 |

## 组织管理

组织管理首发只做单组织，但底层支持扩展。

公司壳默认部门：

```text
总裁办
产品部
研发部
运营部
生活管家部
```

不要一开始放太多部门。财务、法务、市场、销售、客服等可作为后续模板。

组织管理功能：

```text
编辑组织名称
编辑老板称呼
管理部门
管理角色
管理成员归属
设置默认成员模板
设置默认技能包
设置部门可用资产
设置任务分配规则
查看组织工作记录
```

## 资产管理

资产是智能体能使用的资源。首发固定五类：

| 分类 | 说明 | 例子 |
|---|---|---|
| 大脑 | 模型资源 | 本地 Qwen、Ollama 模型、OpenAI-compatible endpoint |
| 账号 | 平台账号和登录凭证 | GitHub、小红书、抖音、邮箱 |
| 钱包 | 支付、链上或资金相关资产 | 支付账号、ETH 钱包、交易所账号 |
| 硬件 | 本地设备和智能家居 | 摄像头、灯、空调、Home Assistant 设备 |
| 知识库 | 文件、笔记、项目资料 | 项目文档、PDF、网页收藏 |

资产管理的核心不是把密码存起来，而是让系统知道：

```text
有什么资源
谁能用
能做什么
怎么用
用之前是否要确认
用完怎么审计
```

## 任务管理

任务管理负责把聊天中的目标变成可追踪的工作。

任务状态：

```text
待规划
规划中
等待确认
执行中
暂停
已完成
失败
取消
已归档
```

任务详情必须显示：

```text
任务目标
成功标准
执行模式
负责人
协作成员
关联资产
关联 Skill
步骤列表
审批记录
工具调用
输出工件
错误与重试
最终回复
记忆写入记录
trace 下载
```

## 系统管理

系统管理固定，不受壳影响。

建议二级菜单：

| 菜单 | 功能 |
|---|---|
| 模型设置 | 本地模型、云模型、路由策略、成本限制 |
| Skill 包 | 安装、启停、版本、评测、权限 |
| MCP 服务 | 服务配置、连接状态、工具列表 |
| 插件 | 插件目录、签名、升级、卸载 |
| 记忆设置 | 写入策略、可视化、导出、清理 |
| 安全策略 | 风险矩阵、确认规则、敏感目录、DLP |
| 日志与审计 | 任务日志、工具日志、审批日志、模型调用 |
| 备份恢复 | SQLite、向量库、配置、Skill 包备份 |
| 壳与外观 | 切换壳、主题、菜单映射 |
| 开发者设置 | API token、调试模式、trace 级别 |

## 典型场景

### 场景 1：日常助理

用户说：

```text
把我今天要做的事整理一下，顺便按优先级排一下。
```

系统行为：

1. 识别为任务规划。
2. 检索最近任务、日历、未完成事项。
3. 小曜整理清单。
4. 输出优先级、建议时间块、可执行下一步。
5. 写入任务计划记忆。

### 场景 2：技术方案

用户说：

```text
帮我把这个项目做成技术方案。
```

系统行为：

1. 小曜作为主持人拆解任务。
2. 宁宁补产品目标和边界。
3. 阿珩设计架构和模块。
4. 小曜汇总成方案。
5. 生成文档工件。
6. 任务详情可回放协作过程。

### 场景 3：账号运营

用户说：

```text
墨白，帮我写一篇小红书草稿，用主账号风格。
```

系统行为：

1. 判断墨白是否能访问小红书主账号。
2. 通过 Asset Broker 获取账号摘要和风格记忆，不暴露明文密码。
3. 调用内容创作 Skill。
4. 生成草稿。
5. 如果要发布，进入确认。

### 场景 4：壳切换

用户把公司壳切到宗门壳。

系统行为：

1. 菜单和字段标签改变。
2. 系统管理保持不变。
3. 资产二级分类保持不变。
4. 成员、部门、岗位的值不自动改变。
5. 用户可手动把“技术经理”改成“炼器长老”。

## 产品验收标准

| 模块 | MVP 验收 |
|---|---|
| 聊天 | 能流式回复，能按场景组织结构，界面无组织/壳信息 |
| 成员 | 能创建至少一个成员，并绑定大脑和人格 |
| 组织 | 公司壳下能管理部门和成员归属 |
| 资产 | 五类资产可创建、编辑、启停、授权 |
| 任务 | 聊天可生成任务，任务有状态和回放 |
| 记忆 | 偏好和任务经历可跨会话召回 |
| Skill | Skill 包可安装、启停、触发 |
| MCP | MCP 服务可配置并暴露工具列表 |
| 安全 | 删除、外发、支付、系统修改等高风险动作必须确认 |
| 系统管理 | 模型、Skill、MCP、记忆、安全、日志可管理 |
