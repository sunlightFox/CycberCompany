# 第五十六阶段 - 长期记忆检索与经验沉淀闭环

## 阶段背景

Hermes 这类 agent 最容易被用户记住的不是“会说话”，而是“它真的记得我、记得项目、记得上次怎么做的”。我们当前有 Memory、Context Gateway、Knowledge 和 trace 的基础，但跨会话记忆抽取、记忆冲突处理、经验复用和主动沉淀还不够像一个可长期依赖的系统。

本阶段继续只做后端、schema、migration、repository、service、API、tests、evals 和文档；不新增前端页面、组件、样式、Tauri 窗口或桌面端交互代码。

## 参考结论

### OpenClaw 采用点

OpenClaw 强调工作空间、上下文和任务状态的持续性：

```text
任务上下文能复用
工作区知识能沉淀
不同会话之间可找回曾经做过的事
```

本项目采用：

```text
任务结果、用户偏好和项目知识进入长期记忆层
记忆检索受 Capability Graph 和 scope 约束
```

### Hermes Agent 采用点

Hermes 的记忆思路更接近“会自己长大”：

```text
记忆分层
偏好长期保留
经验能反向影响后续决策
```

本项目采用：

```text
semantic / episodic / procedural 三层记忆继续分工
成功、失败和纠错经验都能进入可检索的记忆条目
```

## 核心目标

本阶段完成后，后端应支持：

```text
长期记忆抽取、评分、去重和过期治理
跨会话偏好、项目背景和经验复用
任务完成后自动沉淀经验和可复用步骤
记忆冲突、过期和敏感度分层处理
记忆检索可解释、可追踪、可回放
```

## 阶段原则

1. 记忆只记录有价值的事实，不把噪音无限放大。
2. 记忆写入必须有 source。
3. 敏感信息优先脱敏和分层，不进入过宽上下文。
4. 记忆不是任务状态机，不允许反向改写任务事实。
5. 用户可要求遗忘、修正和优先级调整。

## 阶段范围

### 本阶段必须完成

```text
长期记忆写入判定与评分
用户偏好、项目经验和技能经验沉淀
语义检索与时间检索联合策略
记忆冲突、过期和版本演进治理
任务完成后的经验回写和 trace 关联
```

### 本阶段不做

```text
不把所有聊天都强制写入长期记忆
不把敏感信息默认长期保存
不把记忆当成任务执行结果
不绕过用户删除或更正请求
```

## 主要待补

```text
记忆抽取器质量
跨会话偏好复用
经验沉淀回流
记忆冲突治理
长期记忆评测
```
