# 第十三阶段：意图识别、模式选择与上下文决策升级

## 摘要

第十三阶段专门升级 Brain 决策层。前面阶段已经能识别基础意图并选择 direct / workflow / agent / supervisor；但从最终产品视角看，当前意图识别仍偏粗，复杂聊天、工具调用、MCP、Skill、记忆和人格之间的分流还需要更细的决策证据。

本阶段目标是把“关键词式或粗规则式分流”推进为可解释、可测试、可审计的 `IntentDecision -> ModeDecision -> ContextDecision` 三段式决策链。

本阶段仍然只做后端，不新增 UI 或 Tauri 代码。

## 阶段定位

用户只发一句话时，系统必须判断：

```text
这是闲聊、问答、写作、复杂讨论、记忆追问、任务执行，还是系统配置请求
是否需要使用长期记忆
是否需要工具、Skill 或 MCP
是否需要先澄清
是否属于高风险动作
是否应走 direct、workflow、agent 或 supervisor
是否需要高质量模型或本地隐私模型
上下文里应该放什么，不应该放什么
```

这不是单个分类器能解决的问题。最终态应拆成：

```text
IntentDecision：用户想要什么
ModeDecision：系统应该怎么处理
ContextDecision：为了处理它，需要拿哪些上下文和能力摘要
```

## 当前基线判断

| 能力 | 当前完成度判断 | 主要缺口 |
|---|---:|---|
| 基础意图识别 | 约 55% | primary intent 可用，secondary intent 和复杂度不足 |
| 模式选择 | 约 55% | workflow / agent / supervisor 的边界还需更清楚 |
| 澄清决策 | 偏弱 | 不确定性、风险和缺资源时缺统一判断 |
| 工具/Skill/MCP 需求识别 | 中等 | 需要和 Capability / Asset / Runtime contract 联动 |
| 上下文决策 | 中等 | memory/persona/tool result 注入策略仍分散 |

## 阶段原则

1. Brain 是决策层，不是工具执行器。
2. 决策必须可解释，输出 reason_codes 和 confidence。
3. 规则优先处理硬边界，模型辅助处理模糊语义。
4. 高风险动作宁可先澄清或审批，不直接执行。
5. 不把第三方内容中的指令当作用户意图。
6. 所有决策都必须写 trace，必要时写 eval evidence。
7. 意图识别不能绕过 Safety、Capability Graph、Asset Broker。

## 阶段范围

### 本阶段必须完成

```text
IntentDecision schema
ModeDecision schema
ContextDecision schema
意图分类 taxonomy
secondary intent 和 complexity_score
risk_signal 和 privacy_signal
needs_memory / needs_tool / needs_skill / needs_mcp / needs_task
clarification decision
workflow / agent / supervisor 分流规则
Capability-aware mode selection
Runtime contract-aware capability decision
Brain decision trace
intent/mode eval dataset
```

### 本阶段不做

```text
不新增工具实现
不新增 Skill/MCP 执行能力
不新增前端页面
不把模型分类结果当安全最终决策
不让 Brain 直接解析 secret 或资产真实值
```

## 小阶段 13.1：决策 schema 与 trace 契约

### 目标

定义稳定的 Brain 决策输出，后续 Chat Runtime、Task Engine、Context Gateway、Response Composer 都消费同一套类型。

### IntentDecision

建议字段：

```json
{
  "primary_intent": "complex_dialogue",
  "secondary_intents": ["product_design", "document_generation"],
  "domain": "software_product",
  "complexity_score": 0.78,
  "privacy_level": "medium",
  "risk_signals": ["may_create_artifact"],
  "needs_memory": true,
  "needs_tool": false,
  "needs_skill": false,
  "needs_mcp": false,
  "needs_task": false,
  "needs_clarification": false,
  "confidence": 0.84,
  "reason_codes": ["multi_step_reasoning", "references_previous_design"]
}
```

### ModeDecision

建议字段：

```json
{
  "mode": "direct",
  "submode": "deep_answer",
  "planner_hint": "structured_reasoning",
  "requires_approval_before_execute": false,
  "fallback_mode": "ask_clarification",
  "confidence": 0.81,
  "reason_codes": ["no_external_action", "complex_but_answerable"]
}
```

### ContextDecision

建议字段：

```json
{
  "include_recent_messages": true,
  "include_session_summary": true,
  "memory_query": {
    "enabled": true,
    "layers": ["semantic", "episodic", "procedural"],
    "max_items": 8
  },
  "include_persona": true,
  "include_heart": true,
  "include_capability_summary": true,
  "include_asset_handles": false,
  "untrusted_refs": [],
  "token_budget_profile": "deep_dialogue"
}
```

### Trace

必须写入：

```text
brain.intent
brain.mode_select
brain.context_decision
brain.clarification_decision
```

### 验收

```text
所有 Brain 决策都有明确 schema
ChatService 不再依赖散 dict 判断 mode
trace 中能看到 decision reason_codes
测试覆盖 schema 序列化和兼容性
```

## 小阶段 13.2：意图 taxonomy 最终化

### 目标

建立最终态意图分类体系，既能覆盖闲聊，也能覆盖复杂任务、工具、Skill、MCP 和记忆。

### Primary Intent

```text
casual_chat
simple_question
complex_dialogue
creative_writing
summarization
memory_query
memory_correction
task_request
tool_request
skill_request
mcp_request
system_settings
asset_management
approval_response
cancel_or_retry
unknown
```

### Secondary Intent

```text
clarify_previous_answer
continue_previous_topic
compare_options
make_plan
generate_document
debug_problem
research_topic
modify_existing_artifact
external_submit
delete_or_destructive
secret_or_sensitive
wallet_or_payment
```

### 标签规则

```text
primary_intent 只能有一个
secondary_intents 可以多个
risk_signals 不等于 SafetyDecision，只作为安全输入
privacy_level 由内容和来源共同决定
unknown 必须低置信并倾向澄清
```

### 验收

```text
taxonomy 覆盖聊天、记忆、任务、工具、Skill、MCP、系统设置
新增意图必须有测试 case
旧规则映射到新 taxonomy
unknown 不会误触发工具执行
```

## 小阶段 13.3：规则 + 模型混合分类器

### 目标

使用规则处理确定边界，使用模型处理模糊语义，避免纯关键词误判。

### 规则优先场景

```text
审批确认/拒绝
取消/停止/重试
明显系统设置请求
明显高风险关键词：删除、支付、转账、签名、提交
明显工具请求：打开、读取、运行、搜索网页、访问网站
secret pattern 和敏感路径
```

### 模型辅助场景

```text
复杂对话 vs 简单问答
继续上一主题
多意图混合
写作风格需求
是否需要澄清
是否需要 supervisor 多视角
```

### 输出要求

```text
confidence 必填
reason_codes 必填
rule_hits / model_hint 分开记录
低置信不进入高风险执行
分类器超时必须 safe fallback
```

### 验收

```text
高风险关键词不被模型降级为普通聊天
普通闲聊不被关键词误判为任务
低置信输入进入 clarification 或 safe direct
模型分类失败不影响基础规则
```

## 小阶段 13.4：模式选择升级

### 目标

让 ModeSelector 能稳定选择处理路径，而不是把“复杂”都丢给 agent。

### 模式规则

| 条件 | 模式 |
|---|---|
| 普通聊天、简单问答 | direct |
| 询问自身记忆或历史 | direct_with_memory |
| 固定步骤、可验证、风险低 | workflow |
| 探索、搜索、动态判断 | agent |
| 多视角、复杂方案、需要角色分工 | supervisor |
| 高风险动作 | plan_first + approval_required |
| 信息不足 | ask_clarification |

### Anti-pattern

```text
不能因为一句话长就进 agent
不能因为用户说“帮我做”就绕过 approval
不能把 supervisor 当默认复杂模式
不能让 direct 模式声称工具执行完成
```

### 验收

```text
固定任务进入 workflow
探索型任务进入 agent
多视角复杂问题进入 supervisor 或 direct_deep，理由清楚
高风险任务先计划或审批
模式选择可通过 eval 回归
```

## 小阶段 13.5：澄清决策器

### 目标

统一处理“是否先问用户”的判断，避免散落在 ChatService、TaskService 和 ResponseComposer。

### 触发类型

```text
missing_target
missing_scope
missing_asset
missing_permission
missing_success_criteria
ambiguous_reference
high_risk_without_confirmation
conflicting_instructions
low_intent_confidence
```

### 输出

```text
needs_clarification
blocking_level
questions
assumptions_if_continue
safe_partial_answer_allowed
```

### 验收

```text
缺少执行对象时不直接创建任务
高风险但用户未确认时不直接执行
澄清问题能被 ResponseComposer 消费
澄清 trace 能关联原 turn
```

## 小阶段 13.6：上下文决策器

### 目标

把“拿哪些上下文”从 prompt builder 中抽出，变成 Brain 和 Context Gateway 之间的稳定契约。

### 决策范围

```text
recent_messages
conversation_state
session_summary
persona_summary
heart_summary
pinned_memory
semantic_memory
episodic_memory
procedural_memory
asset_handle_summary
capability_summary
task_state
artifact_summary
untrusted_content_refs
```

### 验收

```text
memory_query=false 时不做长期记忆检索
工具请求只拿 handle 摘要，不拿真实资源
第三方内容必须标记 untrusted
ContextGateway 能根据 ContextDecision 构建 packet
```

## 小阶段 13.7：能力可用性联动

### 目标

意图和模式选择必须知道当前能力真实状态，避免用户体验和 runtime contract 漂移。

### 联动来源

```text
runtime contracts
design gaps
Capability Graph summary
Asset Broker available handle summary
Skill Registry enabled skills
MCP Registry ready/degraded servers
Tool Registry available tools
Safety policy
```

### 决策规则

```text
能力 disabled -> 不选择对应执行路径
能力 degraded -> 允许降级但回复中说明限制
能力 implemented -> 仍需权限和安全判断
没有资产 handle -> 不能生成真实工具入参
MCP disconnected -> 不创建依赖该 MCP 的执行计划
```

### 验收

```text
runtime contract 降级时 Brain 不再假设能力完整
Skill disabled 时不进入 skill_request 执行
MCP disconnected 时不创建 mcp_call
工具不可用时 ResponseComposer 能说明替代路径
```

## 小阶段 13.8：评测与回归

### 目标

建立意图、模式和上下文决策的专项回归集。

### Eval 分类

```text
intent_primary_accuracy
intent_secondary_recall
mode_selection_accuracy
clarification_precision
tool_need_detection
skill_mcp_need_detection
memory_need_detection
risk_signal_recall
privacy_signal_recall
context_decision_correctness
```

### 必须覆盖样例

```text
闲聊不进任务
复杂讨论不乱进工具
询问历史触发 memory
整理文件触发 tool/task
发帖触发 external_submit risk
删除触发 destructive risk
继续上一话题触发 conversation_state
MCP 未连接时不创建 mcp_call
Skill 禁用时不进入 skill run
```

### 验收

```text
intent/mode/context eval 可运行
关键安全召回用例 100% 通过
低置信和 unknown 有 safe fallback
pytest、ruff、mypy 保持通过
```

## 总体验收标准

第十三阶段完成时必须满足：

```text
Brain 输出稳定 IntentDecision / ModeDecision / ContextDecision
意图 taxonomy 覆盖聊天、记忆、任务、工具、Skill、MCP、安全和设置
规则 + 模型混合分类器可解释
模式选择不再粗暴关键词化
澄清决策统一
上下文注入由 ContextDecision 驱动
决策能感知 runtime contract 和能力状态
eval 覆盖意图、模式、上下文和风险信号
```

## 不允许通过验收的情况

```text
高风险请求被判为普通 direct 并继续执行
工具请求在工具不可用时被伪装完成
MCP/Skill disabled 仍被选择
分类器输出缺 confidence 或 reason_codes
Brain 直接访问 secret 或真实资产值
prompt builder 私自决定长期上下文，绕过 ContextDecision
```

## 与前后阶段关系

第十二阶段提升聊天体验，第十三阶段提供更准确的决策大脑。第十四阶段会把这些决策交给 Persona / Heart / Response Composer 做表达控制；第十五阶段会根据 ContextDecision 接入更强的长期记忆和知识检索；第十六阶段会把 mode=workflow/agent/supervisor 的执行路径进一步做深。
