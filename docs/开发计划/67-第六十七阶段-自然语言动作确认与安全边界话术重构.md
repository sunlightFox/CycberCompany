# 第六十七阶段 - 自然语言动作确认与安全边界话术重构

## 阶段背景

自然语言确认、拒绝、修改、取消和安全边界是聊天执行体验的核心。当前 `natural_chat.py`、`chat_quality.py`、`chat_safety.py` 与 `chat.py` 中仍有多处旧硬编码话术和补丁逻辑，能覆盖一部分场景，但整体语义还不够统一。

本阶段目标是把所有动作确认与安全边界可见回复统一到 pending-action 语义：用户说人话即可确认、拒绝或修改；系统必须说明真实状态，不暴露技术 ID，不把等待确认或失败说成完成。

## 核心目标

完成后应满足：

```text
确认、拒绝、修改、取消都绑定明确 pending action
一句模糊“好的”不能释放多个高风险动作
没有 pending action 时不能假装确认成功
跳过审批、越权登录、假装执行必须 fail closed
等待确认、执行中、失败、取消、完成的可见话术统一
所有可见回复不泄露 approval_id / task_id / tool_call_id / trace_id
```

## 破坏性调整

### pending-action v4 语义

统一结构：

```json
{
  "scenario": "natural_interaction",
  "natural_interaction": {
    "status": "pending_action|approved|denied|edited|no_pending_action|blocked",
    "reason_codes": [],
    "pending_actions": [],
    "reply_options": [],
    "clear_pending": false
  },
  "response_quality_guard": {
    "state_disclosed": true,
    "boundary_disclosed": true,
    "next_step_provided": true,
    "no_false_done": true,
    "no_internal_terms": true
  }
}
```

旧字段可兼容读取，但新写入以 v4 为准。

### 可见话术统一

所有动作类文本从统一 catalog 取：

```text
action.pending
action.approved_progress
action.approved_completed
action.approved_failed
action.denied
action.edited
action.no_pending
action.ambiguous_blocked
action.multiple_pending
boundary.approval
boundary.privacy
boundary.persona
boundary.internal
boundary.professional
boundary.desktop
```

## 实现要求

### 1. 重构 `NaturalChatActionGateway`

影响范围：

```text
apps/local-api/app/services/natural_chat.py
apps/local-api/tests/test_phase34_natural_chat_interaction_loop.py
apps/local-api/tests/test_phase52_chat_deploy_install.py
```

要求：

```text
handle() 只负责解析自然语言控制意图和 pending action 绑定
_resolve_action() 只处理 approve / deny / edit，不生成散落文案
所有结果通过 ResponseComposer 生成可见文本
visible_text_guard 继续过滤技术字段
多 pending action 时必须要求用户点名
高风险动作禁止“永久允许”
```

### 2. 重构 `ChatQualityPolicy`

影响范围：

```text
apps/local-api/app/services/chat_quality.py
apps/local-api/tests/test_phase41_chat_quality_experience.py
apps/local-api/tests/test_xiaowu_chat_quality.py
```

必须覆盖：

```text
最新指令优先
桌面能力边界
安全拒绝
真人/隐藏账号边界
系统 prompt / trace 拒绝
专业建议边界
隐私高输入可恢复阻断
```

所有边界回复都必须：

```text
承接用户意图
说明不能做什么
说明可以替代做什么
不创建任务
不调用工具
不生成 approval
```

### 3. 重构 `ChatVisibleOutputFilter`

影响范围：

```text
apps/local-api/app/services/chat_safety.py
apps/local-api/app/services/chat_response.py
```

要求：

```text
内部 ID 替换稳定
工具 XML 泄漏直接剔除
模型安全标记不可见
风险等级转成自然描述
浏览器证据补充不污染普通回复
严格格式不被安全补丁破坏
```

### 4. 清理 `chat.py` 中补丁话术

要求：

```text
chat.py 不新增用户可见硬编码长文案
主链路只负责调用 service 和 composer
任务、审批、取消、失败、隐私、质量边界都交给专门服务
```

## 验收标准

```text
确认下载只释放当前唯一下载 pending action
没有 pending action 时回复“没有等待确认的动作”，不说已完成
拒绝后说明动作已停，不继续执行
修改参数只修改原 action payload，不串线
跳过审批和假装执行不创建任务、不生成 approval
系统 prompt 请求给替代帮助，不泄露内部内容
可见回复中无技术 ID 和内部字段
```
