# 第八十九阶段 - 聊天质量误拦截治理与规则减法

## 阶段定位

系统现在不是“不会防守”，而是有些时候防守太多了。  
本阶段专门治理误拦截、误澄清、误边界回复，把大量补丁型启发式判断收缩到真正该保留的硬边界上。

## 阶段目标

```text
压缩 deterministic reply 与启发式捷径
把“像是分析请求 / 像是继续 / 像是改口”从正则直答改成主链路判断
只保留硬边界 deterministic guard
建立误拦截率和误澄清率评测
恢复聊天自然度而不牺牲安全
```

## 核心问题

当前存在两类典型体验损伤：

```text
正常分析请求被边界回复吞掉
正常延续对话被误判成澄清或安全提醒
运行时错误有时直接暴露成用户可见失败文案
```

## 本阶段范围

### 必须完成

```text
盘点并分类现有 deterministic / heuristic 分支
只保留真正 fail-closed 的 deterministic guard
其余转成 brain / context / state machine 的正规判断
建立误拦截与误澄清 regression suite
```

### 明确不做

```text
不放松注入、私网、元数据地址、本地敏感路径等硬边界
不删除审批与安全主链
不改渠道可靠性逻辑
```

## 允许保留的 deterministic guard

只允许以下类型继续保留 deterministic reply 或 hard block：

```text
metadata / private network / unsupported scheme 边界
明确 prompt injection / secret exfiltration 边界
不可恢复或高危动作 hard block
已有 pending approval / pending clarification 的显式状态响应
严格格式输出保护
```

## 优先迁出的 heuristic

以下类型不应再由简单启发式直接终结主链：

```text
像是 plain analysis request
像是 latest instruction override
像是 short followup
像是 explicit continuation
像是 vague clarification followup
```

它们应改由：

```text
SessionContext
ContextGateway
BrainDecision
pending state machine
ResponseComposer
```

共同决定。

## 实施拆解

### 89.1 heuristic inventory

目标：

```text
把所有 _looks_like_* / _deterministic_* / quick reply fallback 盘点清楚
```

交付：

```text
heuristic registry
每个 heuristic 的 owner / risk / replace target
保留名单与删除名单
```

### 89.2 硬边界与软启发式分离

目标：

```text
把必须 fail-closed 的 guard 和“只是猜测用户意图”的 shortcut 分开
```

交付：

```text
hard boundary guard list
soft heuristic deprecation list
替代主链落点
```

### 89.3 主链判断回收

目标：

```text
把 continuation / latest instruction / plain analysis 等判断回收到正式主链
```

交付：

```text
state-aware continuation logic
current-message-priority override logic
brain-assisted or rules-first but non-terminal routing
```

### 89.4 误拦截评测门禁

目标：

```text
让误拦截从“体感问题”变成量化问题
```

交付：

```text
false boundary rate
false clarification rate
natural continuation pass rate
runtime failure visible leakage count
```

## schema / migration

