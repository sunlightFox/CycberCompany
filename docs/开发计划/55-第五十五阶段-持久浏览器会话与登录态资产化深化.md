# 第五十五阶段 - 持久浏览器会话与登录态资产化深化

## 阶段背景

第四十七、第五十阶段已经把浏览器持久执行、外部平台 Provider 插件化和无开放 API 的浏览器操作适配打出框架，但真实用户更关心的是“能不能长期复用我授权过的浏览器会话，少登录、少断线、少重来”。目前浏览器工具链虽然可执行，但在 profile 资产、登录态恢复、页面状态续航和跨任务复用上还需要再硬化一层。

本阶段继续只做后端、schema、migration、repository、service、API、tests、evals 和文档；不新增前端页面、组件、样式、Tauri 窗口或桌面端交互代码。

## 参考结论

### OpenClaw 采用点

OpenClaw 对浏览器与执行容器的理解很务实：

```text
执行上下文可复用
浏览器 profile 和 workspace 分开管理
不同任务之间有明确隔离边界
```

本项目采用：

```text
把浏览器会话当成受控资产，而不是一次性工具状态
profile、cookie、local storage 和 page state 分层管理
```

### Hermes Agent 采用点

Hermes 的价值在于多执行后端和持久会话体验：

```text
同一任务链路可以复用长会话
浏览器、终端和外部平台状态能连续推进
失败后能回到上一个可继续的位置
```

本项目采用：

```text
浏览器 session 与 task checkpoint 绑定
登录态只作为可审计资产，不作为明文秘密扩散
```

## 核心目标

本阶段完成后，后端应支持：

```text
持久 browser profile 资产和 session asset
登录态复用、失效检测和受控恢复
browser.open/click/fill/submit 的连续执行状态
页面证据、截图、DOM 摘要和网络摘要可回放
浏览器会话与账号资产、能力图和任务 trace 联动
```

## 阶段原则

1. 登录态是资产，不是普通参数。
2. profile 只能在授权范围内复用，不能跨成员串用。
3. 浏览器失败要给出具体状态，不允许沉默重试到假成功。
4. 页面证据优先，不能只返回“已完成”。
5. 浏览器访问外部站点默认受安全边界、URL 策略和任务绑定约束。

## 阶段范围

### 本阶段必须完成

```text
browser profile / browser session asset schema
session 恢复、过期、重连和失效诊断
页面状态、截图、网络摘要、console 证据统一回放
浏览器操作与 task checkpoint 绑定
外部站点登录态资产化与 redaction
```

### 本阶段不做

```text
不绕过平台验证码或安全提示
不把浏览器 profile 明文交给模型
不把无权限 URL 当成可访问页面
不新增桌面 UI
```

## 主要待补

```text
长期 profile 目录管理
登录态健康探测
页面状态回放
任务级浏览器证据收敛
跨任务会话复用策略
```
