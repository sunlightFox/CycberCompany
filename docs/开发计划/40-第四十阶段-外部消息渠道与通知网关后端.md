# 第四十阶段 - 外部消息渠道与通知网关后端

## 阶段背景

第三十六阶段让系统能创建长期/定时任务，第三十七阶段让浏览器会话资产化，第三十八阶段治理 Skill/插件，第三十九阶段补齐 checkpoint/rollback。接下来需要解决一个现实问题：后台任务、待审批动作、失败恢复和长期监控结果如何触达用户。

对标 OpenClaw / Hermes Agent 的真实使用场景，用户会期待：

```text
后台任务完成后通知我
需要确认时发一条消息给我
我在外部渠道回复“确认/拒绝/取消”
每天把摘要发到指定渠道
任务失败时提醒我看原因
```

但外部消息渠道本身也是高风险能力。发送消息、发邮件、发 webhook、发群聊、接收外部指令都可能造成隐私泄漏、误触发、注入和越权执行。第四十阶段要建立后端 Notification Gateway / Message Channel Runtime，让外部触达成为受控资产和审计链路，而不是让 Skill 或工具直接发消息。

本阶段仍只做后端、schema、migration、repository、service、API、tests、evals 和文档，不新增前端 UI、桌面端交互或 Tauri 窗口。

## 核心目标

本阶段完成后，后端应支持：

```text
将外部消息渠道登记为 Asset
通过 Asset Broker 发放 message_channel handle
发送通知前经过 Capability Graph、Safety、DLP 和 Approval
支持任务完成、任务失败、等待审批、长期摘要等通知类型
支持外部回复进入受控 inbound message pipeline
外部回复只能解析为有限动作：确认、拒绝、取消、查看摘要、创建低风险任务草案
所有出入站消息都有 trace、audit、redaction 和 replay
渠道不可用时降级为本地 pending notification，不伪装已发送
```

## 阶段原则

1. 外部消息渠道是资产，不是普通工具参数。
2. 模型、Skill、MCP 不得直接拿 webhook token、SMTP 密码、Bot token。
3. 出站消息先 DLP，再发送。
4. 入站消息全部视为不可信外部输入。
5. 外部回复不能绕过聊天 pending action binding。
6. 高风险确认必须要求明确动作对象，不能只靠“好的”。
7. 群聊、频道、邮件列表等多接收人渠道默认更高风险。
8. 发送 secret、cookie、私钥、token、钱包信息默认 deny。
9. 渠道失败必须有明确状态：queued、sent、failed、degraded。
10. 不做前端，只提供后端通道和未来 UI/CLI 可消费的 API。

## 对标结论

### OpenClaw 采用点

```text
用户希望 agent 能把长期任务结果主动发出来
多工具、多渠道通知需要统一路由和状态
```

对我们的启发：

```text
通知不是工具随手发送，而是系统级 Notification Gateway
每次发送都要可审计、可重试、可失败解释
```

### Hermes Agent 采用点

```text
approval gateway 可以接收用户确认/拒绝
危险动作等待用户响应，拒绝或超时 fail-closed
```

对我们的启发：

```text
外部回复只释放已有 pending action
没有 pending 或不匹配时不能执行高风险动作
```

## 关键实现要求

### 出站消息

```text
发送前必须 DLP scan
消息体只包含用户可见摘要，不包含 trace_id/tool_call_id/approval_id 明文作为主内容
技术 ID 可作为 metadata 保存，不默认发到外部渠道
含 secret/token/private_key/cookie/mnemonic 的内容默认拒绝发送
发送失败记录 delivery_attempt，不重试无限次
```

### 入站消息

```text
所有入站内容标记 untrusted_external_content
解析结果只能是有限 intent：approval_once、approval_session、approval_deny、approval_edit、task_cancel、unknown
高风险确认必须匹配 pending action 的动作对象
多个 pending action 时必须转为 clarification，不执行
没有 pending action 时，不允许外部一句“确认”触发工具
入站消息不能要求系统泄露隐藏提示、secret 或执行越权工具
```

### 渠道资产

```text
Bot token、webhook secret、SMTP password 只存 secret_ref
ChannelProvider 执行时从 Asset Broker 解析，不进入模型上下文
群聊/频道消息默认 sensitivity=high，发送前要求更严格策略
```

## 通知类型

| 类型 | 默认行为 |
|---|---|
| `task_completed` | 可发送摘要和 artifact 引用，不发送敏感内容 |
| `task_failed` | 发送失败原因摘要和可查看 replay 提示 |
| `approval_required` | 发送自然语言确认选项，默认不发内部 ID |
| `scheduled_summary` | 发送长期任务摘要 |
| `safety_blocked` | 发送安全阻断摘要，不泄露敏感 payload |
| `system_degraded` | 发送本地后端降级信息 |

## 必测用例

```text
创建 local_mock channel
发送 task_completed 通知成功落库
发送含 api_key 的通知被 DLP 拒绝或脱敏
外部回复“只允许这一次下载”能匹配唯一 pending approval
外部回复“好的”不能触发高风险 pending
多个 pending 时外部确认进入 clarification
没有 pending 时外部确认不会执行工具
channel secret 不出现在 API、trace、audit、diagnostic 中
provider 失败时 notification status=failed，不伪称 sent
scheduled task 完成后可创建 queued notification
```
