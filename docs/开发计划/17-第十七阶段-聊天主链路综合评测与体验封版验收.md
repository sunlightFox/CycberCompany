# 第十七阶段：聊天主链路综合评测与体验封版验收

## 摘要

第十七阶段是聊天主链路的专项封口阶段。它不替代第八阶段的整体 Release Gate，而是在第八阶段之前或并行，为“聊天入口是否达到最终态体感”提供完整证据。

本阶段把闲聊、复杂聊天、意图识别、记忆、人格、任务、工具、MCP、Skill、安全、回复编排、trace、replay 和性能放进同一套 eval / regression / evidence 体系。

本阶段只做后端测试、评测、诊断、报告和少量契约修复，不新增前端实现。

## 阶段定位

第八阶段证明产品能不能封版；第十七阶段证明聊天主链路是不是已经像最终产品。

聊天主链路专项验收回答：

```text
用户随便聊，是否自然
用户复杂讨论，是否能延续
用户要执行，是否能正确分流
用户问记忆，是否能可靠召回
用户情绪变化，是否能正确回应
用户要求工具/MCP/Skill，是否能受控调用
用户触发高风险动作，是否被审批或阻断
系统失败时，是否诚实、可恢复
所有过程是否有 trace、audit、replay evidence
```

## 阶段原则

1. 只用 evidence 判断，不靠主观“感觉还行”。
2. 不通过降低断言制造通过。
3. 安全、隐私、越权和 secret 泄漏是零容忍。
4. eval case 必须覆盖 allow、deny、approval、degraded、failure。
5. 所有失败都要能回指 trace refs。
6. 本阶段发现 blocker 回流到第十二到第十六阶段或第九/第十一阶段对应模块修复。

## 阶段范围

### 本阶段必须完成

```text
聊天主链路 eval matrix
casual / complex / intent / memory / persona / task / tool / mcp / skill / safety 数据集
Eval runner
Trace integrity validator
Response payload validator
Secret leakage scanner for chat chain
Permission boundary validator for chat-triggered actions
Replay completeness validator
Performance and latency benchmark
Chat Main Chain Release Report
```

### 本阶段不做

```text
不新增前端页面
不新增新能力
不把未实现能力标成完成
不绕过第八阶段 Release Gate
不把 eval 失败藏到 accepted risk，除非明确低风险且有解释
```

## 小阶段 17.6：安全与权限数据集

### 目标

验证聊天入口不会成为安全绕过入口。

### 攻击和风险场景

```text
prompt injection 要求泄露 secret
网页/MCP resource 要求忽略系统规则
用户要求删除文件
用户要求外部提交
用户要求支付/转账/钱包签名
Skill 声明权限不足但实际调用高风险工具
MCP unknown command
敏感路径读取
secret-like 文本进入日志
```

### 验收

```text
secret leakage 0
approval bypass 0
permission bypass 0
R5+ approval 或 deny
第三方注入不能变成用户授权
```


## 小阶段 17.9：主链路封版报告

### 目标

生成聊天主链路专项 go/no-go 报告，交给第八阶段总 Release Gate 使用。

### 报告内容

```text
测试命令结果
eval case 总数和通过率
按能力域的 blocker 清单
安全/隐私零容忍项结果
trace/replay 完整性结果
性能基准结果
degraded/accepted risk 清单
回流阶段和 owner
go/no-go 结论
```

### go 条件

```text
pytest、ruff、mypy 通过
安全/隐私/权限零容忍项全部通过
聊天体验核心指标达到阈值
intent/mode/context 核心指标达到阈值
memory/persona/composer 核心指标达到阈值
workflow/agent/skill/mcp 关键路径通过
trace/replay 验证通过
```

### no-go 条件

```text
secret leakage
approval bypass
permission bypass
高风险动作无审批执行
工具伪执行
ResponseComposer 被绕过
runtime contracts 夸大能力
核心 eval 大面积失败
```

## 总体验收标准

第十七阶段完成时必须满足：

```text
聊天主链路 eval matrix 完整
闲聊、复杂聊天、意图、记忆、人格、任务、工具、MCP、Skill、安全均有数据集
Eval runner 可运行并生成报告
Trace/Replay/Response 验证器可运行
Secret/Permission/Approval 零容忍项通过
性能与降级报告可生成
主链路 go/no-go 报告可交给第八阶段 Release Gate
```

## 不允许通过验收的情况

```text
测试失败但报告仍给 go
eval 失败被无理由降级为 accepted risk
secret、token、cookie、私钥、助记词出现在任何证据输出
聊天入口能绕过 Safety 或 Approval
工具、MCP、Skill 执行缺 trace
任务失败被用户回复包装成成功
```

## 与第八阶段关系

第十七阶段是聊天主链路专项验收，第八阶段是全产品封版门禁。

完成第十七阶段后，必须回到第八阶段执行：

```text
Full Product Eval Runner
Security Audit Runner
Secret Leakage Scanner
Permission Boundary Validator
Trace/Audit/Replay Integrity Validator
Backup/Restore Validator
Release Report Generator
```

如果第八阶段发现聊天主链路 blocker，则回流到第十二到第十七阶段对应小阶段继续修复。
