# 第一百零七阶段 - 长期记忆语义契约统一与纠错闭环硬化

## 阶段定位

phase92 已经把长期记忆的 recall、freshness、cross-session preference 召回和 correction 流程搭出来了，但契约仍然有零散漂移，尤其是 superseded、filtered reason、纠错后的旧记忆下线语义，还没有完全稳定。

这一阶段聚焦“记忆结果到底怎么解释”，把 search / filtered / recall_scope / correction 的返回语义彻底拉齐。

## 目标

```text
把记忆 recall / supersede / correction 的返回契约彻底统一
让 superseded 等状态枚举稳定
让 cross-session preference recall 排序与过滤原因稳定
让记忆纠错后的旧记忆可解释下线
```

## 重点

### 107.1 状态枚举统一

```text
superseded、expired、archived、freshness_state 等枚举收口
filtered reason 与最终状态字段不再混用不同命名
```

### 107.2 recall 与 filtered 结果一致

```text
search 结果中的 items、filtered、recall_scope_applied 语义一致
同一条旧记忆为何被排除要可解释且可复现
```

### 107.3 correction 闭环硬化

```text
记忆纠错后，新记忆上线、旧记忆下线、superseded_by 关系保持稳定
cross-session preference recall 应优先返回纠正后的新事实
```
