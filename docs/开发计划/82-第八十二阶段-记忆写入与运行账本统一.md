# 第八十二阶段 - 记忆写入与运行账本统一

## 阶段定位

本阶段目标是把聊天主链路中的：

```text
turn
tool call
approval
task handoff
memory write
trace span
```

统一成一套可追溯、可审计、可回放的运行账本。

## memory write 的 source 结构

任何长期记忆写入都必须带 `source`，结构固定为：

```json
{
  "type": "conversation_turn | tool_result | task_result | approval_resolution | external_ingest",
  "conversation_id": "conv_xxx",
  "turn_id": "turn_xxx",
  "message_id": "msg_xxx",
  "task_id": "tsk_xxx",
  "tool_call_id": "tool_xxx",
  "approval_id": "apr_xxx",
  "trace_id": "trc_xxx",
  "channel": "local | wechat | feishu",
  "captured_at": "ISO8601"
}
```

### 强约束

```text
source 中允许记录 trace_id 作为内部关联键
但 trace_id 默认不进入用户可见回复
source 不能缺 conversation_id 与 captured_at
```

## turn ledger / run ledger / trace span 关联键

### turn ledger

最小字段：

```text
turn_id
conversation_id
session_id
member_id
trace_id
status
route_type
mode
started_at
ended_at
retry_of_turn_id
recovered_from_turn_id
```

### run ledger

面向一轮执行展开事件：

```text
run_id
turn_id
trace_id
stage
event_type
status
ref_id
ref_type
summary
created_at
```

`ref_type` 固定值：

```text
tool_call
approval
task
memory_write
artifact
channel_delivery
recovery_attempt
```

### trace span

继续承担底层时序与性能/故障证据，但需要能被 ledger 反向索引。

最小要求：

```text
每个 ledger 事件都能定位相关 trace span
每个关键 span 都能回链到 turn_id
```

## direct turn、tool turn、task handoff turn 的记忆写入条件

### direct turn

允许写入：

```text
稳定用户偏好
长期关系信息
被显式确认的重要事实
可复用的方法性经验摘要
```

禁止写入：

```text
临时 open loop
本轮未确认的推测
工具尚未执行的计划
```

### tool turn

允许写入：

```text
明确成功的只读查询结论
可复用的工具使用经验
用户偏好的执行边界
```

禁止写入：

```text
原始工具输出全文
高风险动作的未确认结果
暂时性执行状态
```

### task handoff turn

允许写入：

```text
task 已创建这一事实
task 完成后的经验总结
稳定流程经验
```

禁止写入：

```text
尚未完成的 task 结果
后台临时中间态
未验证的外部证据
```

## 用户可修正、可归档、可追溯契约

必须支持：

```text
查看记忆来源
按 source_turn_id 追踪写入来源
归档错误记忆
用 supersede 方式修正旧记忆
保留修正链，不直接抹除审计轨迹
```

### 最小字段要求

每条长期记忆至少有：

```text
memory_id
member_id
conversation_id
source
memory_type
status
supersedes_memory_id
created_at
updated_at
```

## 诊断包与时间线回放字段

诊断包最少包含：

```text
turn summary
route summary
tool/approval/task timeline
memory write decisions
redaction summary
failure / recovery decisions
artifact refs
```

### 时间线回放

必须能按以下顺序回放：

```text
user input
context build
brain decision
safety decision
tool or task branch
response finalize
memory write
channel delivery
```

## 审计要求

以下行为必须有 audit event：

```text
高风险工具执行
approval 请求与批准/拒绝
memory 写入、归档、修正
channel peer 绑定或重绑
跨渠道 identity link
```

## 验收标准

```text
所有长期记忆写入都有 source
turn/tool/approval/task/memory 都能串到统一 ledger
回放可定位关键 trace span
错误记忆可以 supersede，而不是无痕覆盖
诊断包能说明“为什么写入/为什么没写入”
```

## 最小测试集

```text
direct turn 写入偏好记忆时包含 source
tool turn 写入经验记忆时包含 tool_call_id 与 trace_id
task handoff 不会提前把未完成结果写入长期记忆
memory 修正生成 supersede 链
diagnostic bundle 能回放 turn -> tool -> memory write 时间线
audit 中不泄漏 secret / token / cookie
```
