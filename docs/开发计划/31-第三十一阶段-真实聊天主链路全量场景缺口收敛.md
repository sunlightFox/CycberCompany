# 第三十一阶段：真实聊天主链路全量场景缺口收敛

## 摘要

第三十一阶段承接第三十阶段的真实聊天主链路 E2E 修复闭环，把 `CHAT-E2E-20260429` 从基础批次扩展到九轮全量产品测试后的开发收敛阶段。

截至当前测试目录 `docs/测试/聊天主链路/2026-04-29/`，已完成基础、扩展、深度、稳定性回归、恢复一致性、知识总结、多维场景、任务执行、浏览器专项九轮测试：

```text
总用例：152
通过：116
失败：36
阻塞：0
待修复问题：69
```

全量测试证明：基础闲聊、真实模型预检、显式非敏感记忆、敏感记忆拒写、Skill/MCP 基础链路、任务生命周期、终端和文件边界、浏览器 open/snapshot/screenshot/download、流式事件与 replay 已有可用证据；但真实用户期望下的知识总结质量、浏览器搜索/登录交互、系统任务证据、意图路由、记忆纠错、Persona 边界和敏感信息治理仍存在缺口。

本阶段目标不是新增 UI，而是把这些真实测试暴露的问题纳入正式开发阶段：先修 P0 安全泄漏，再补 P1 主链路能力，再收敛 P2 稳定性和质量问题，最后通过全量真实测试和阶段回归证明闭环。

本阶段只做后端、schema、migration、repository、service、API、tests、evals、diagnostic 和报告，不新增前端页面、组件、样式、Tauri 窗口或桌面交互代码。

## 阶段定位

第三十一阶段回答：

```text
真实聊天主链路是否能覆盖用户日常闲聊、知识总结、复杂聊天和任务执行
浏览器搜索、模拟登录、点击、填表、提交和截图是否能形成可审计任务证据
用户问知识时是否默认得到全面、有结构的总结，而不是任务完成模板
用户明确说不执行、不创建任务、不调用工具时，系统是否稳定尊重
密码、token、cookie、私钥、本地敏感路径是否在回复、任务、工具、trace、报告中全链路脱敏
记忆纠错、遗忘、召回、Persona/Heart 和 session 隔离是否在多轮真实聊天中稳定
工具、终端、文件、浏览器任务的错误语义、审批状态和工件证据是否可回放
全量真实测试是否能进入 release report、diagnostic bundle 和阶段验收门禁
```

## 当前基线判断

| 能力 | 当前状态 | 缺口 |
|---|---|---|
| 真实模型聊天 | 可用 | 部分知识/解释型问题被误路由到 workflow，缺少 `model.completed` |
| 闲聊与复杂聊天 | 基础可用 | 最新指令覆盖、长上下文压缩、边界诚实仍有失败 |
| 知识总结 | 多数通过 | 浏览器自动化证据总结等场景被当成任务执行，缺关键词和结构 |
| 记忆与人格 | 基础可用 | 记忆纠错/遗忘误判、内部 trace 字段暴露、Persona 边界不稳定 |
| 会话隔离 | 部分可用 | 同一 conversation 下不同 `session_id` 主题可能串话 |
| 任务执行 | 基础可用 | 不执行约束可能被忽略，删除审批拒绝状态机失败 |
| 工具边界 | 多数可用 | 未知工具、无任务绑定终端、错误码语义仍需收敛 |
| 浏览器能力 | 只读能力可用 | 缺少 search/click/fill/type/submit，搜索和登录无法真实完成 |
| 浏览器任务证据 | 部分可用 | 聊天回复缺少搜索结果、页面标题、截图、snapshot、artifact 证据 |
| 敏感信息治理 | 基础 DLP 可用 | 登录测试密码在任务证据链仍有泄漏风险，文件名存在误脱敏 |
| 回归证据 | 有九轮报告 | 69 个问题需要映射到开发票据和阶段验收 |

## 阶段原则

1. 不通过降低断言、删除失败用例或伪造回复来让测试变绿。
2. 安全泄漏优先于功能补齐，P0 修复必须先落地并补回归测试。
3. 用户问知识、解释概念、要求总结时，默认走直接回答和真实模型，不误建任务。
4. 用户要求执行浏览器、终端、文件、系统任务时，必须经过 Task Engine、Safety、Approval、Tool Runtime 和 Trace。
5. 用户明确说“不执行”“不要创建任务”“不要使用工具”“只生成方案”时，必须尊重最新指令。
6. 浏览器和系统任务不能只回复“已完成”，必须返回可核验的页面、结果、工件、错误或恢复建议。
7. 记忆写入必须有 source，对外记忆查询不得暴露内部 trace、turn、message 定位字段。
8. Persona/Heart 只能改善语气，不能覆盖安全边界，不能声称真人或隐藏执行能力。
9. 本阶段所有新增能力继续遵守 Asset Broker、Capability Graph、Safety、Approval 和 Trace 边界。

## 与第三十阶段的关系

第三十阶段解决基础真实聊天 E2E 批次暴露的 4 个首轮缺口，重点是把真实聊天从 deterministic eval 拉回可复现、可修复、可诊断的闭环。

第三十一阶段在第三十阶段基础上扩大到九轮、152 条真实产品测试，重点从“首轮 E2E 修复”推进到“全量场景缺口收敛”：

```text
第三十阶段：基础真实 E2E 缺口修复与封版实测闭环
第三十一阶段：全量真实场景、浏览器/系统任务、知识总结、记忆人格和安全证据收敛
```
