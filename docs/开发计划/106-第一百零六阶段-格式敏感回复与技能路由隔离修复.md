# 第一百零六阶段 - 格式敏感回复与技能路由隔离修复

## 阶段定位

当前聊天主链路已经具备 direct answer、office/skill 路由、结构化输出和工具执行能力，但格式敏感请求仍然会被技能话术、工具计划或渠道兼容逻辑污染。

这一阶段解决的不是“能不能回答”，而是“当用户明确要求 JSON、表格、纯文本等输出格式时，系统能不能守住格式边界，不把别的执行域话术串进来”。

## 目标

```text
解决 JSON、表格、纯文本等格式请求被技能/工具路由污染的问题
让 format-sensitive 请求优先级稳定高于花哨技能话术
把 direct answer 与 office / skill 路由解耦
统一 response plan plain_text / structured payload 的行为
```

## 重点

### 106.1 format-sensitive 请求优先级

```text
“只输出 JSON”“输出表格”“只给纯文本结果”这类意图要先被识别
格式约束优先于技能推荐语气、office 工具引导话术和泛化执行建议
```

### 106.2 direct answer 与技能路由隔离

```text
direct answer 可以独立完成时，不应被 office / skill 路由二次污染
skill/tool 只有在真正承担执行责任时才介入回复计划
```

### 106.3 结构化输出一致化

```text
response plan 中 plain_text 与 structured payload 的来源一致
JSON / markdown table / plain_text 的最终成品与计划声明一致
```
