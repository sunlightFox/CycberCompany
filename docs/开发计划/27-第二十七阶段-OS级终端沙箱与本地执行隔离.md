# 第二十七阶段：OS 级终端沙箱与本地执行隔离

## 摘要

第二十七阶段聚焦“终端和本地执行的硬隔离”。当前第二十一阶段已经实现 ToolActionPolicy、CommandRiskClassifier、TerminalSandboxProfile、OutputDLP 和执行边界诊断；但 `OSLevelSandbox`、`TerminalRunner` 仍为 degraded，实际隔离是 `none_with_policy_guard`。

本阶段目标是在 Windows 本地环境中引入更强的进程级和文件系统级隔离，把终端执行从策略护栏推进到 OS 级受限执行。

本阶段只做后端，不新增 UI。

## 阶段定位

第二十七阶段回答：

```text
terminal.run 是否能在 OS 限权进程中执行
工作目录是否被强制限制在 task artifact sandbox
环境变量、网络、输出大小、运行时间是否受控
危险命令是否在进入进程前被阻断
进程执行是否可中断、可清理、可审计
降级到 policy guard 时是否如实标注
```

## 当前基线判断

| 能力 | 当前状态 | 缺口 |
|---|---|---|
| TerminalSandboxProfile | implemented | 仅 policy profile |
| CommandRiskClassifier | implemented | 规则风险分级已有 |
| OutputDLP | implemented | 输出脱敏已有 |
| OSLevelSandbox | degraded | 无 Job Object / low integrity / container |
| TerminalRunner | degraded | task artifact policy guard，无 OS 隔离 |

## 阶段原则

1. 终端执行必须绑定 task。
2. 默认工作目录必须是 task artifact sandbox。
3. 环境变量默认最小化，不继承敏感 env。
4. R5+ 命令必须 approval 或 deny。
5. OS sandbox 不可用时必须降级并写 evidence。
6. 输出必须限长、脱敏、落 artifact。

## 阶段范围

### 本阶段必须完成

```text
Windows Job Object runner
低权限进程 profile
环境变量最小化
文件系统路径 allow/deny
进程超时和 kill tree
输出流限长与 DLP
资源使用记录
sandbox eval
```

### 本阶段不做

```text
不允许任意 cwd
不允许模型绕过 ToolRuntime 执行命令
不默认开放网络写操作
不新增 UI
不自动删除用户目录文件
```

## 小阶段总览

| 小阶段 | 名称 | 核心交付 |
|---:|---|---|
| 27.1 | Sandbox backend 抽象 | policy_guard/job_object/low_integrity/container |
| 27.2 | Windows Job Object runner | 进程组、timeout、kill tree、资源限制 |
| 27.3 | 低权限环境与 env policy | 最小环境、secret env 阻断 |
| 27.4 | 文件系统边界 | sandbox root、sensitive denylist、path resolver |
| 27.5 | 网络与外发限制 | network profile、external submit approval |

## 小阶段 27.1：Sandbox backend 抽象

### 目标

让 TerminalRunner 通过 backend interface 执行，而不是直接依赖当前实现。

### Backend

```text
policy_guard
windows_job_object
windows_low_integrity
container
disabled
```

### 接口

```text
prepare(profile)
run(command, cwd, env, timeout)
stream_output()
terminate()
cleanup()
diagnostics()
```

### 验收

```text
backend 状态可查询
不可用 backend 自动降级 policy_guard
降级原因写 sandbox diagnostics
```

## 小阶段 27.2：Windows Job Object runner

### 目标

在 Windows 上限制进程树、运行时间和资源使用。

### 限制项

```text
kill_on_job_close
process_time_limit
wall_time_limit
memory_limit
active_process_limit
breakaway_denied
```

### 验收

```text
超时进程被杀死
子进程随 job 关闭清理
资源使用进入 tool_call result
失败不泄漏系统路径
```

## 小阶段 27.3：低权限环境与 env policy

### 目标

让终端执行不继承敏感环境变量。

### Env policy

```text
clear_by_default
allowlist_keys
deny_secret_patterns
inject_task_context_only
no_user_profile_env
```

### 验收

```text
API_KEY/TOKEN/SECRET/COOKIE 不被继承
工具需要的 env 只能通过 env_ref/asset handle
env policy 写入 boundary decision
```

## 小阶段 27.4：文件系统边界

### 目标

从 cwd 限制推进到路径解析和文件系统边界。

### 路径策略

```text
allowed_roots = task artifact dirs
readonly_roots = explicitly granted workspace dirs
denied_roots = user home/system/secrets/wallet/browser profiles
symlink_policy = deny_escape
relative_path_policy = normalize_and_check
```

### 验收

```text
路径穿越失败
符号链接逃逸失败
敏感路径读取失败
artifact 写入成功
```

## 小阶段 27.5：网络与外发限制

### 目标

把终端网络动作从普通命令中分离出来。

### 网络 profile

```text
offline
loopback_only
trusted_read
external_read
external_write_requires_approval
```

### 验收

```text
curl/wget/powershell web request 被识别
external_write 需要 approval
支付/钱包/登录类动作 deny 或 approval
```

## 小阶段 27.6：Sandbox 评测

### 必测 case

```text
timeout kill tree
env secret not inherited
path traversal deny
symlink escape deny
system path deny
artifact write allow
network write approval
dangerous command deny
degraded fallback evidence
```

### 验收命令

```text
.venv\Scripts\python.exe -m pytest apps\local-api\tests\test_phase27_os_sandbox.py
.venv\Scripts\python.exe -m ruff check .
.venv\Scripts\python.exe -m mypy .
```

## 阶段总验收标准

第二十七阶段完成时必须满足：

```text
TerminalRunner 支持至少一个 OS 级 sandbox backend
不可用时可降级且 evidence 明确
终端执行 env、cwd、path、timeout、output 全受控
R5+ 不可能无审批执行
sandbox eval 覆盖 escape、timeout、secret、network
```
