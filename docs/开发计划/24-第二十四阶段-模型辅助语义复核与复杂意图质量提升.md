# 第二十四阶段：模型辅助语义复核与复杂意图质量提升

## 摘要

第二十四阶段聚焦“复杂对话和意图识别的真实语义质量”。当前第十八阶段已经落地 `DialogueStateService`、`SemanticIntentAnalyzer` 和 `LowConfidenceDecisionReviewer`，能够跟踪目标、约束、转向、多意图、冲突和低置信度 review；但 `ModelAssistedVerifier` 仍为 `disabled`，低置信度复核只记录规则 fallback。

本阶段目标是在严格受控边界内启用模型辅助语义复核，让复杂意图、上下文冲突、隐含任务、高风险含糊表达和跨轮引用不再完全依赖关键词规则。

本阶段只做后端，不新增 UI。

## 阶段定位

第二十四阶段不是替换规则链路，而是在规则链路之上增加“只读、建议型、可关闭”的模型复核层。

它回答：

```text
规则判断低置信度时，是否能调用模型复核
模型复核是否只输出结构化建议，不执行动作
多意图、上下文冲突和高风险含糊表达是否更准
模型复核失败时，规则 fallback 是否仍然稳定
模型输出是否经过 schema validation、redaction 和 safety guard
复核质量是否能通过 eval 证明
```

## 当前基线判断

| 能力 | 当前状态 | 缺口 |
|---|---|---|
| DialogueStateService | implemented | 规则优先，语义抽取深度有限 |
| SemanticIntentAnalyzer | implemented | 多意图可拆，但仍偏关键词 |
| LowConfidenceDecisionReviewer | implemented | 有 review 记录，模型复核未启用 |
| ModelAssistedVerifier | disabled | 只有 seam，没有真实模型调用 |
| 复杂意图 eval | implemented | 当前是 deterministic local smoke |

## 阶段原则

1. 模型复核只能读上下文摘要，不能拿 secret、真实路径、token、私钥、cookie。
2. 模型复核只生成 `SemanticReviewSuggestion`，最终决策仍由 BrainDecisionService 归一化。
3. 模型复核不能创建任务、不能调用工具、不能写记忆、不能审批。
4. 模型复核必须可配置关闭；关闭后规则路径必须全量通过。
5. 高风险场景中，模型建议只能提高澄清/审批概率，不能降低风险等级。
6. 所有模型复核都必须写 trace、usage、latency、schema validation 和 fallback evidence。

## 阶段范围

### 本阶段必须完成

```text
SemanticReviewRequest / SemanticReviewSuggestion schema
ModelAssistedVerifierService
低置信度触发策略
模型输出 schema validation
模型输出 risk monotonic guard
模型复核结果融合策略
模型不可用 fallback
复核质量 eval
runtime contract 状态更新
```

### 本阶段不做

```text
不让模型直接执行工具
不让模型直接写记忆
不让模型直接修改 SafetyDecision
不把模型复核作为唯一决策来源
不新增前端
不做云端强依赖；本地无模型时必须降级
```

## 小阶段 24.1：复核 schema 与触发策略

### 目标

定义模型复核的输入、输出、触发条件和 fallback 规则。

### SemanticReviewRequest

```text
review_id
turn_id
conversation_id
member_id
redacted_user_text
dialogue_state_summary
semantic_candidate_summary
intent_decision_summary
mode_decision_summary
context_decision_summary
capability_boundary_summary
risk_signal_summary
privacy_level
trigger_reasons
trace_id
```

### SemanticReviewSuggestion

```text
suggested_primary_intent
suggested_secondary_intents
suggested_mode
missing_information
ambiguous_references
context_conflicts
risk_notes
memory_notes
tool_notes
skill_notes
mcp_notes
clarification_questions
confidence
reason_summary
```

### 触发条件

```text
intent confidence < threshold
mode confidence < threshold
semantic conflicts 非空
用户引用上文但 dialogue state 置信度不足
同时命中 memory/tool/skill/mcp/task
高风险动作缺少对象、范围、目的地或确认
能力不可用但用户明确要求执行
```

### 验收

```text
所有 request 字段都经过 redaction
trigger_reasons 可查询
preview 模式不产生持久副作用
无可用模型时 fallback_used=true
```

## 小阶段 24.2：ModelAssistedVerifierService

### 目标

实现受控模型复核服务，默认可关闭，可配置启用。

### 服务职责

```text
选择允许的 brain
构造最小化 review prompt
调用模型
解析 JSON
schema validation
去除内部推理标签
过滤危险建议
记录 usage 与 trace
```

### 模型选择约束

```text
privacy high -> local_only
allow_cloud=false -> local_only
无 healthy local brain -> fallback
超时 -> fallback
schema invalid -> fallback
```

### 验收

```text
模型复核不接收 raw secret
模型返回非法 JSON 不影响主链路
模型输出不能包含 tool call 指令
复核结果写入 low_confidence_review
```

## 小阶段 24.3：决策融合与风险单调性

### 目标

把模型建议安全地融合到 BrainDecision，不让模型降低安全边界。

### 融合规则

```text
规则和模型一致 -> 提高 confidence
模型指出缺失信息 -> clarification
模型指出高风险 -> risk_level 不低于原始判断
模型认为可执行但规则认为高风险 -> 保持高风险
模型建议工具/Skill/MCP 但能力不可用 -> capability_boundary
模型建议记忆写入但 privacy high -> memory_vs_privacy conflict
```

### 风险单调性

```text
R3 不能被模型降为 R1/R2
approval_required 不能被模型改为 allow
deny 不能被模型改为 allow
SafetyDecision 不受 persona/heart/model verifier 降级影响
```

### 验收

```text
高风险缺目的地仍问澄清
能力不可用仍不创建任务
模型复核不能绕过 Safety/Approval
融合结果有 reason_codes
```

## 小阶段 24.5：复杂意图评测集增强

### 目标

### 必测 case

```text

高风险支付/转账缺目的地
用户要求绕过审批
模型输出非法 JSON
模型超时 fallback
privacy high local_only
```

### 指标

## 小阶段 24.6：Runtime contract 校准

### 目标

如实更新模型复核能力状态。

### 状态规则

```text
未配置模型 -> implemented_with_fallback
配置本地模型并通过 eval -> implemented
只配置云模型但 privacy guard 未通过 -> degraded
schema/fallback 测试失败 -> blocker
```

### 验收

```text
ModelAssistedVerifier 不再长期 disabled
runtime contracts 标注 real_model_call、fallback、privacy_policy
design gaps 保留可接受风险和复查条件
release report 纳入 phase24
```

## 阶段总验收标准

第二十四阶段完成时必须满足：

```text
低置信度语义复核可真实调用模型或稳定 fallback
模型复核输出只作为结构化建议
风险等级、审批、拒绝不能被模型降级
复杂意图 eval 覆盖多意图、上下文冲突、高风险和 fallback
runtime contracts 如实标注 implemented / implemented_with_fallback / degraded
```
