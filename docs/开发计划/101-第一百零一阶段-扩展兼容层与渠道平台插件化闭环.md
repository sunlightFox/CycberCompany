# 第一百零一阶段 - 扩展兼容层与渠道平台插件化闭环

## 阶段定位

前面的能力建设已经把 Skill、MCP、渠道、外部平台这些能力铺开了，但核心运行时里仍然存在一个越来越明显的问题：

```text
第三方 Skill / plugin 导入路径不统一
bundled 渠道和平台实现仍带有宿主硬编码痕迹
兼容格式、绑定状态、MCP 同步、执行前规划还没有形成统一闭环
```

第一百零一阶段要把“扩展”升级成一等后端能力，目标不是只支持安装一个插件，而是形成：

```text
preview -> import -> canonicalize -> install -> enable -> bind -> plan-run -> replayable evidence
```

## 阶段目标

```text
建立 extension canonical package 统一抽象
把 OpenClaw skill / plugin 导入收口到统一兼容层
让渠道扩展和外部平台扩展通过标准 definition / registry 接入，而不是继续向核心层泄漏具体实现
打通 preview、compatibility、install、enable、bind、plan-run、MCP 同步和绑定快照
把扩展结果证据接入 migration / repo / service / API / tests
```

## 本阶段范围

### 必须完成

```text
定义 canonical extension package、compatibility report、binding snapshot 契约
支持 OpenClaw skill v1 与 OpenClaw plugin v1 的导入、预览、安装和兼容性评估
建立 extension_packages / extension_sources / extension_compatibility_reports / extension_binding_snapshots 数据落盘
建立 ChannelExtensionDefinition / ExternalPlatformExtensionDefinition 契约与 registry
把 bundled wechat、feishu、fake、browser、xiaohongshu 改成扩展注册，而不是核心层硬编码分支
提供 /api/extensions/preview-import、/install、/enable、/disable、/bind、/plan-run
支持启用阶段的 MCP server 注册与同步
```

### 明确不做

```text
不在本阶段建设前端插件市场或图形化管理界面
不把所有第三方生态格式一次性吃完，优先收口 OpenClaw 兼容路径
不把 provider 不可用、MCP 未就绪或绑定缺项伪装成 fully ready
不让扩展绕过 Asset Broker、Capability Graph、Safety、Approval、Trace 等核心边界
```

## 实施拆解

### 101.1 Canonical 兼容导入层

目标：

```text
让异构扩展先进入统一 canonical package，而不是在业务层到处兼容
```

交付：

```text
canonical_extension_package
compatibility_report
permission_preview
canonical_snapshot
```

### 101.2 渠道与平台扩展注册表

目标：

```text
让 bundled 渠道和外部平台通过标准扩展契约接入核心运行时
```

交付：

```text
channel_extension_registry
external_platform_extension_registry
provider_inventory
route_mounts
seeded_targets
```

### 101.3 安装启用绑定闭环

目标：

```text
把 enable、bind、plan-run、MCP 同步和绑定证据做成统一扩展生命周期
```

要求：

```text
启用后扩展状态、兼容性与绑定缺项必须可查询
MCP server 注册 disabled / ready 状态要诚实呈现
instruction-only skill 与需要 MCP 的 plugin 要有不同 runnable 语义
plan-run 结果必须能区分 matches、runnable、warnings
```

```text
扩展从“分散的插件兼容逻辑”升级为统一后端执行能力
系统能稳定输出 canonical package、兼容性报告、绑定快照、MCP 同步状态和 plan-run 语义

```
