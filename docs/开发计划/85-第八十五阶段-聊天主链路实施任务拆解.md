# 第八十五阶段 - 聊天主链路实施任务拆解

## 阶段定位

本阶段不是定义新架构，而是把第七十六到第八十四阶段的规格真正拆成可执行批次，明确：

```text
每一批先改哪些模块
每一批允许接管什么
每一批不允许碰什么
每一批最小测试集是什么
```

## 实施总原则

```text
先收口 runtime，再统一 session/context，再补 tool loop，再治理 response/ledger/hook
每一批都必须保持当前聊天体验不回退
每一批都优先新增中层与契约，不先暴力删除兼容层
任何批次都不能绕过 Asset Broker / Capability Graph / Safety / Trace
```

### 明确禁止



### 回退边界

```text

允许 Skill / MCP / plugin 先经 hook bridge 接入而不是直接全量迁移

```

```text

hook 面稳定可扩展
```
