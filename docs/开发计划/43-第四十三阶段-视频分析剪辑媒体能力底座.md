# 第四十三阶段 - 视频分析剪辑媒体能力底座

## 阶段背景

第三十四阶段把聊天入口收敛为自然语言文字交互：用户只说目标，系统优先进入可执行链路，在危险点阻塞确认。第四十三阶段在这个主入口之上，补齐“视频分析和剪辑”所需的后端媒体能力底座。

当前代码已经具备任务、工件、文件工具、浏览器截图/下载、终端沙箱、Skill/MCP 插件、审批和 trace，但视频还不是一等公民：

```text
聊天输入只支持 text
attachments 只保存元数据，不进入媒体处理链路
工具注册没有 media.* 或 video.*
模型 adapter 只发送文本 messages
没有 ffmpeg / ffprobe 能力探测
没有抽帧、音频转写、场景检测、剪辑计划和渲染服务
```

本阶段参考两个方向：

| 参考对象 | 采用点 | 不直接照搬 |
|---|---|---|
| OpenClaw | media understanding、video generation、provider adapter、异步任务和 artifact 契约 | 不把云视频生成当作默认能力，不绕过本地优先和资产权限 |
| Hermes Agent | vision fallback、browser/terminal/code tools、Skills 生态组合媒体流程 | 不把视频能力只做成散落脚本，必须进入稳定 schema、service、trace 和 eval |

最终目标是让系统具备可审计的视频分析和基础剪辑能力，并为未来接入 ComfyUI、Whisper、Veo、Sora、Runway、Qwen、fal 等外部媒体 provider 预留稳定契约。

## 核心目标

第四十三阶段完成后，后端应能支持：

```text
导入或引用任务工件中的视频、音频和图片
对视频做 ffprobe 元信息分析
从视频抽帧并生成低成本视觉摘要
提取音频并转写成带时间戳文本
做基础场景切分、镜头摘要和时间线索引
把用户自然语言剪辑需求转为结构化 Edit Decision List
用受控 ffmpeg 后端执行裁切、拼接、转码、抽音频、合字幕等基础剪辑
所有媒体处理都有 task、tool_call、artifact、approval、trace 和 replay 证据
Skill/MCP 可以接入更高级的媒体 provider，但不能绕过 Asset Broker、Safety 和 Approval
```

本阶段只做后端、schema、migration、repository、service、API、tests、evals、diagnostic 和文档，不新增前端页面、组件、样式、Tauri 窗口或桌面端交互。

## 阶段原则

1. 媒体必须成为后端一等契约，不把视频处理藏在通用 `terminal.run` 里。
2. 本地优先：基础分析和基础剪辑默认使用本地 ffmpeg/ffprobe 或等价本地后端。
3. 云 provider 可选：外部视频理解、生成和重绘必须作为 Asset/Tool/MCP/Skill 接入。
4. 大文件只进 artifact，不进模型上下文；模型只拿摘要、帧索引、时间线和低风险预览。
5. 视频原始内容、抽帧、转写、剪辑结果必须有 checksum、content_type、size、source 和 trace。
6. 剪辑、转码、覆盖、导出、上传和云处理都要经过 Safety、Approval 和 DLP。
7. 不把未安装 ffmpeg、缺少模型或缺少 provider 包装成已完成能力。
8. 不让 Skill 直接读取任意本地视频路径；输入必须来自授权 artifact、asset handle 或受控下载。
9. 媒体工具输出默认标记 untrusted 或 derived，不允许覆盖用户指令和安全策略。
10. 所有新增公共服务和高风险路径必须有最小测试；涉及 trace、审批、资产权限必须补覆盖。

## 对标结论

### OpenClaw 路线

OpenClaw 的媒体路线更偏“媒体一等能力”：

```text
Media Understanding 节点先识别附件类型
image/audio/video/document 分流到可用模型或 CLI fallback
视频理解结果被压缩成可注入的 [Video] 摘要块
原附件仍保留，供后续工具或模型使用
video_generate 作为 provider tool 暴露 generate / imageToVideo / videoToVideo
异步 provider 任务完成后回写会话和 artifact
工具只有在 provider 可用时才注册或展示
```

对我们的启发：

```text
先做 media understanding pipeline，再做剪辑工具
视频分析结果必须有结构化 timeline，不只是一段自由文本
provider adapter 要被能力发现、资产权限、成本策略和 trace 管住
异步视频任务要能暂停、恢复、失败解释和回放
```

### Hermes Agent 路线

Hermes Agent 的媒体路线更偏“通用工具 + Skill 生态”：

```text
图片和截图进入 vision-capable 模型或 vision_analyze fallback
浏览器工具可产生 screenshot / accessibility snapshot / visual analysis
terminal、process、execute_code 可调用本地脚本和外部工具链
Creative/Media 类 Skill 把 ComfyUI、Manim、ascii-video 等能力组合为工作流
```

对我们的启发：

```text
基础媒体能力要在 Tool Runtime 内稳定可审计
复杂媒体工作流放到 Skill/MCP，不写死到核心层
视觉 fallback 应作为模型路由的一部分，而不是让普通模型假装看过视频
终端仍可作为底层执行后端，但不能成为用户和 planner 看到的唯一媒体契约
```

## 小阶段 43.1：媒体 schema、migration 与 artifact 派生关系

### 目标

建立媒体对象和工件派生关系，让视频、音频、抽帧、转写、剪辑结果都能被后端稳定索引、追踪和回放。

### 实现要求

```text
新增 schemas/media.py，定义 MediaAsset、MediaDerivative、MediaAnalysis、EditPlan
新增 migration：media_assets、media_derivatives、media_analysis、media_edit_plans
TaskArtifact 增加或通过 metadata 表达 media_role、parent_artifact_id、derived_from
repository 封装媒体 CRUD，不在 service 中散落 SQL
artifact URI 仍使用 task artifact 沙箱，不保存任意本地路径
所有媒体记录必须有 source_artifact_id、checksum、created_at、trace_id 或可追溯 source
```

### 验收

```text
测试库从零 migration 成功
可把一个 fixture mp4 登记为 media asset
可查询 media asset 的源 artifact、派生 artifact 和分析记录
切换壳不改变媒体底层数据值
```

## 小阶段 43.2：ffmpeg/ffprobe 后端与能力诊断

### 目标

实现受控媒体执行后端，避免 planner 或 Skill 直接拼 `terminal.run ffmpeg ...`。

### 实现要求

```text
新增 MediaRuntime 或 MediaToolService，封装 ffmpeg/ffprobe 调用
启动或诊断时探测 ffmpeg、ffprobe 路径、版本、支持的编码器
执行 cwd 固定在任务 artifact 目录，拒绝自定义 cwd 和路径逃逸
命令由结构化参数生成，不接受用户自由 shell 字符串
执行有 timeout、max_output_bytes、max_input_size、max_duration_seconds
stdout/stderr 进入 terminal_log 或 media_log artifact，并经过 redaction/DLP
ffmpeg 缺失时返回明确 MEDIA_BACKEND_UNAVAILABLE，不伪装成功
```

### 验收

```text
无 ffmpeg 环境下 probe/render 返回受控错误
有 ffmpeg 环境下 fixture probe 成功
路径穿越、敏感路径、自定义 cwd、shell injection 均被拒绝
媒体执行日志不泄漏本机敏感路径和环境变量
```

## 小阶段 43.3：视频分析工具链

### 目标

让系统能“看懂视频的结构”，但不把整段视频塞进模型上下文。

### 实现要求

```text
media.probe 返回时长、编码、分辨率、音视频流、旋转、码率等元信息
media.extract_frames 支持 interval、timestamps、scene_boundary 三种模式
media.extract_audio 输出 wav/mp3/m4a artifact，并记录采样率和声道
media.transcribe_audio 先实现契约和 adapter interface，本地可用时接 whisper/faster-whisper，缺失时受控 degraded
media.scene_detect 先提供基础帧差或 ffmpeg scene 阈值方案
media.timeline_summarize 合并 probe、frames、transcript、scenes，输出结构化时间线
所有 frame/transcript/scene/timeline 都写入 media_analysis 或 derivative
```

### 验收

```text
短视频 fixture 可生成 probe、frames、audio artifact
无转写模型时转写返回 degraded，不影响 probe 和抽帧
timeline 至少包含 start_ms、end_ms、summary、evidence_artifact_ids
大视频超过策略限制时返回可解释错误或要求用户确认降采样
```

## 小阶段 43.4：剪辑计划与渲染执行

### 目标

把“帮我剪成 30 秒短视频”转成可审核、可执行、可回放的结构化剪辑计划。

### 实现要求

```text
media.plan_edit 根据用户目标、timeline、transcript 生成 EditDecisionList
plan_edit 只生成计划，不修改媒体文件
EDL 必须通过 schema 校验：时间范围合法、片段不越界、输出 profile 合法
media.render_edit 只接受已校验 EDL，不接受自由 ffmpeg 命令
支持基础操作：trim、concat、crop/scale、mute、audio_keep、subtitle、transcode
输出 artifact_type=video，content_type=video/mp4 或指定安全格式
覆盖已有输出、导出到外部目录、上传外部平台必须进入审批
```

### 验收

```text
fixture 视频可按 EDL 裁切出新 mp4 artifact
非法时间范围、负时长、超出源视频时长被拒绝
render_edit 有 tool_call、approval、trace、artifact、media_derivative 记录
用户要求“只给剪辑方案，不要执行”时只生成 EDL，不渲染
```

## 小阶段 43.5：模型路由与多模态摘要

### 目标

将视频理解拆成模型可承担的轻量任务：帧摘要、转写摘要、时间线推理、剪辑计划，而不是让文本模型假装看过视频。

### 实现要求

```text
ModelRouter 增加 media_understanding、frame_summary、audio_transcription、edit_planning 路由任务类型
supports_vision / supports_audio 真正参与候选过滤
不支持 vision 的模型不能直接处理图片帧，只能读取已生成的文本摘要
frame summary 可走 vision-capable 模型；无可用模型时使用低置信占位并标 degraded
transcript summary 可走文本模型，但必须标明来源是转写文本
Context Gateway 只注入压缩后的 timeline、frame captions 和 transcript chunks
```

### 验收

```text
无 vision 模型时不会声称已经视觉分析，只返回可用 probe/transcript 结论
有 vision mock adapter 时 frame summary 能进入 timeline
隐私 high 的本地视频默认禁止云模型，除非用户明确授权并有资产策略
```

## 小阶段 43.6：Skill/MCP 媒体扩展

### 目标

让高级媒体能力通过 Skill/MCP/Asset 接入，核心层只维护稳定契约和安全边界。

### 实现要求

```text
定义 media provider Skill 模板：用途、输入、输出、步骤、工具、风险、禁止项
支持注册 mcp.comfyui.*、mcp.whisper.*、mcp.video_provider.* 等工具，但默认 disabled
provider API key、endpoint、账号、成本策略必须作为 Asset 或 settings secret 管理
Skill 安装前展示媒体权限：读取哪些 media artifact、是否上传云端、输出类型、成本风险
云 provider 调用必须记录 provider、request_id、成本摘要、输入 checksum，不记录明文密钥
外部 video-to-video / image-to-video / text-to-video 只作为扩展能力，不进入基础验收门槛
```

### 验收

```text
媒体 Skill 包可安装、预览权限、启用、禁用、运行 eval
声明未安装或未授权 provider 时，匹配结果显示缺依赖，不伪装成功
MCP 输出不能直接变成 terminal command 或外部上传动作
```

## 小阶段 43.7：安全、审批、DLP 与 replay

### 目标

保证视频分析和剪辑“能做事，但不能越界”：大文件、敏感内容、云上传、覆盖导出、账号发布都必须有清晰边界。

### 风险模型

| 动作 | 默认处理 |
|---|---|
| probe、读取元信息、读取已授权媒体摘要 | 允许，写 trace |
| 抽帧、抽音频、生成转写 | 允许或轻提示，依敏感度和大小决定 |
| 生成剪辑计划 | 允许，不修改源文件 |
| 渲染新 artifact | R3，通常需要确认或任务内合并确认 |
| 覆盖已有工件、批量生成、大文件高资源任务 | R4/R5，需要确认 |
| 上传到云 provider、外部平台、账号发布 | R4/R5，必须 Asset + Approval |
| 读取任意本地视频路径、访问敏感目录、携带 secret 的 URL | deny/hard_block |

### 实现要求

```text
ActionRiskDecision 覆盖 media.* 动作
render_edit 前绑定 EDL hash、source checksum、output profile、task_id、member_id
执行前再次校验 binding，源文件 checksum 改变则重新确认
DLP 扫描 transcript、frame captions、tool result、logs 和 report
Task replay 展示 media steps、source、derivatives、analysis、edit plan、output artifact
diagnostic bundle 不包含原始视频内容，只包含脱敏元信息和小预览引用
```

### 验收

```text

拒绝审批后不生成输出 artifact

任务回放能解释视频从输入到输出的每个派生步骤
```

## 与第三十四阶段关系

第三十四阶段负责“自然语言文字入口 + 执行优先 + 危险点阻塞”。第四十三阶段把视频分析和剪辑纳入这条入口：

```text
用户说“帮我分析这个视频” -> 创建媒体分析任务 -> probe/抽帧/转写/timeline -> 自然语言总结证据
用户说“剪成 30 秒短视频” -> 生成 EDL -> 等待渲染确认 -> render_edit -> 输出视频 artifact
用户说“只给剪辑方案，不要执行” -> 只生成 EDL，不调用 render_edit
用户说“上传到外部平台” -> 进入 Asset + Safety + Approval，不自动发布
```

本阶段完成后，系统可以诚实地说“我能分析和基础剪辑任务工件中的视频”，但仍不能宣称具备任意云视频生成、实时视频陪伴或自动发布能力。
