# 第十五阶段：长期记忆语义检索与知识上下文升级

## 摘要

第十五阶段专门补齐 Memory / Knowledge 在聊天主链路中的深度能力。当前系统已经具备记忆写入、FTS fallback、source、部分检索和上下文注入基础；但最终态要求不是“能搜到文本”，而是能在长期对话、复杂任务、知识库问答、偏好变化和旧事实冲突中稳定召回正确内容。

本阶段目标是把 `fts_fallback` 降为真正 fallback，让向量检索、时间有效性、supersede、来源解释和上下文选择形成闭环。

本阶段只做后端，不新增 UI。

## 阶段定位

长期记忆是个人智能体操作系统的护城河。聊天主链路中的记忆能力至少要回答：

```text
我是否记得用户稳定偏好
我是否记得项目和任务上下文
我是否知道旧事实可能被新事实替代
我是否能说明记忆来源
我是否能避免把敏感信息错误注入模型
我是否能在向量不可用时清楚降级到 FTS
```

第十五阶段重点连接：

```text
MemoryService
KnowledgeService
VectorStore
ContextGateway
Brain ContextDecision
ResponseComposer memory_notice
Trace / Eval
```

## 当前基线判断

| 能力 | 当前完成度判断 | 主要缺口 |
|---|---:|---|
| Memory Service | 约 70% | 真实语义检索、冲突和时效策略需增强 |
| Knowledge Service | implemented with FTS | 向量索引和 chunk 质量需补齐 |
| VectorStore | degraded | Chroma/provider 仍未成为默认真实链路 |
| Context 注入 | 中等 | selection_reason、temporal/supersede 规则需增强 |

## 阶段原则

1. 记忆写入必须包含 source。
2. 检索必须可解释，返回 selection_reason。
3. 向量不可用时才进入 FTS fallback，不伪装成功。
4. 旧事实不直接覆盖，新事实通过 supersede / valid_from / valid_to 表达。
5. 敏感记忆默认不进入模型上下文，除非策略明确允许。
6. Knowledge indexing 失败不伪装成功。
7. 所有记忆检索、写入、索引、降级都必须 trace。

## 阶段范围

### 本阶段必须完成

```text
VectorStore provider 抽象落地
Chroma provider 或本地 vector provider
disabled provider 与 fts fallback provider
embedding provider 抽象
Memory 写入 embedding + vector upsert
Knowledge chunk + embedding + vector upsert
semantic / episodic / procedural / asset memory 检索融合
temporal / supersede 召回规则
selection_reason
memory_notice
vector unavailable degraded trace
记忆和知识 eval
```

### 本阶段不做

```text
不新增记忆可视化 UI
不做云端同步
不引入重型图数据库作为强依赖
不把所有历史消息无差别向量化
不自动写入高敏感记忆
```

## 小阶段 15.2：Memory 写入向量闭环

### 目标

记忆从写入开始就具备语义检索能力。

### 写入流程

```text
extract candidate
validate source
score confidence / sensitivity
persist memory_item
generate embedding
upsert vector ref
write memory.vector_upsert span
on failure write degraded reason
```

### Vector metadata

```text
memory_id
member_id
organization_id
layer
kind
sensitivity
valid_from
valid_to
supersedes
source_type
created_at
```

### 验收

```text
记忆 source 缺失则拒绝写入
embedding 成功时有 vector_ref
embedding 失败时 memory 仍可保存但标记 degraded
敏感记忆不进入默认上下文
```

## 小阶段 15.3：Knowledge 索引升级

### 目标

让知识库从 FTS 搜索升级为 chunk + vector + source trace 的混合索引。

### Chunk 策略

```text
按文档结构优先切分
保留标题层级
保留 source_uri / artifact_id
保留 checksum
保留 chunk_order
限制 chunk token 上限
```

### 索引流程

```text
ingest document
extract text
chunk
persist chunks
generate embeddings
upsert vector refs
write source trace
mark index status
```

### 验收

```text
知识库索引失败不伪装成功
chunk 有 source 和 checksum
vector 可用时 knowledge search 返回 semantic hits
vector 不可用时明确 fallback 到 FTS
```


```text

ContextGateway 只注入摘要和 refs
```

## 小阶段 15.5：时效、冲突与 supersede

### 目标

避免长期记忆污染当前事实，尤其是偏好、身份、项目状态发生变化时。

### 规则

```text
新事实不硬删除旧事实
冲突事实进入 candidate review 或自动 supersede
valid_to 结束旧事实有效期
用户明确纠错优先级最高
低置信自动写入只进入候选，不直接覆盖高置信记忆
```

### 场景

```text
用户过去喜欢咖啡，现在改喝茶
项目名称变更
默认写作风格改变
某账号/资产失效
用户要求忘记某记忆
```

### 验收

```text
偏好变化能生成 supersede
旧事实不会被默认召回为当前事实
用户纠错有 trace 和 audit
冲突记忆能在 memory_notice 中说明
```

## 小阶段 15.6：ContextGateway 记忆注入

### 目标

让 ContextGateway 按第十三阶段的 ContextDecision 注入记忆，而不是无差别检索。

### 注入规则

```text
needs_memory=false 时不做长期检索
memory_query.layers 控制检索层
max_items 和 token_budget 限制注入
只注入摘要，不注入敏感原文
保留 source_ref 和 selection_reason
第三方知识内容标记 untrusted
```

### 验收

```text
普通闲聊不无谓检索大量长期记忆
记忆追问能召回相关历史
复杂项目讨论能召回项目记忆
注入结果有 memory.search span
```

## 小阶段 15.7：隐私与敏感记忆控制

### 目标

防止记忆系统成为敏感信息泄露通道。

### 敏感等级

```text
public
low
medium
high
secret
wallet
credential
```

### 控制规则

```text
credential/wallet 默认不进入模型上下文
secret 只可作为 handle/ref，不注入明文
高敏感记忆检索需要 policy 允许
trace/audit 记录 memory_id 和 sensitivity，不记录明文
ResponseComposer 必须能输出 memory_notice 或安全阻断
```

### 验收

```text
secret-like memory 不进入 prompt 明文
trace 中不出现私钥、token、cookie、助记词
高敏感记忆召回需要明确 reason
secret leakage scanner 覆盖 memory/context payload
```

### 验收

```text

pytest、ruff、mypy 保持通过
```

## 总体验收标准

第十五阶段完成时必须满足：

```text
VectorStore 真实接入 Memory / Knowledge 链路
向量不可用时才进入 FTS fallback
Memory 写入包含 source、embedding 或 degraded reason
Knowledge chunk 有 source、checksum、vector refs
检索结果有 selection_reason
temporal/supersede 避免旧事实污染
敏感记忆不进入上下文明文
记忆和知识 eval 可运行
```

## 不允许通过验收的情况

```text
fts_fallback 被标为 vector 成功
记忆 source 缺失仍可写入
知识索引失败但状态为 success
被 supersede 的旧事实仍默认作为当前事实回答
secret 明文进入 prompt、trace、audit、response payload
ContextGateway 绕过 ContextDecision 自行检索全部记忆
```

## 与前后阶段关系

第十五阶段为第十二阶段的复杂聊天、第十三阶段的上下文决策、第十四阶段的 memory_notice 和第十六阶段的任务反思提供可靠长期上下文。完成后，聊天主链路才能真正具备“越聊越懂你，但不乱记、不乱泄露”的最终态基础。
