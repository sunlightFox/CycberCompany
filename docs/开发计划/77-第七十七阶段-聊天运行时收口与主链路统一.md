# 第七十七阶段 - 聊天运行时收口与主链路统一

## 阶段定位

本阶段承接第七十六阶段总计划，目标不是新增聊天功能，而是把当前真正承担 turn 执行的代码从 `ChatService + ChatTurnExecutionOrchestrator` 中收束成正式 `ChatRuntime`。

阶段完成后，主链路必须满足：

```text
SessionRuntime 只是外部入口层
ChatRuntime 成为唯一真实 turn runtime
ChatService 降为 façade / 依赖组装层
route handler 与 channel gateway 不再承载业务编排
```

## 当前问题

虽然项目中已经存在：

```text
services/chat-runtime/chat_runtime/runtime.py
apps/local-api/app/services/session_runtime.py
apps/local-api/app/services/chat_turn_execution.py
```

但实际运行态仍然存在以下问题：

```text
共享包 ChatRuntime 仍主要是 placeholder contract
SessionRuntime 只是转发壳层，但没有绑定正式 ChatRuntime
ChatService 仍持有过多 turn 运行时细节
chat_turn_execution.py 只承担局部编排，不能独立代表完整 runtime
```

## 目标边界

### 1. SessionRuntime

`SessionRuntime` 只保留以下外部入口：

```python
class SessionRuntime:
    async def create_turn(self, request: ChatTurnRequest, *, retry_of_turn_id: str | None = None) -> ChatTurnResponse: ...
    async def run_turn(self, turn_id: str) -> None: ...
    async def stream_turn_events(self, turn_id: str) -> AsyncIterator[ChatEvent]: ...
    async def cancel_turn(self, turn_id: str) -> ChatTurnResponse: ...
    async def retry_turn(self, turn_id: str) -> ChatTurnResponse: ...
    async def recover_incomplete_turns(self) -> int: ...
```

职责限定：

```text
只负责公开入口语义
只负责把外部请求转发到 ChatRuntime
保留 runtime diagnostic
不直接执行聊天编排
```

### 2. ChatRuntime

正式主接口定义为：

```python
class ChatRuntime:
    async def create_turn(self, request: ChatTurnRequest, *, retry_of_turn_id: str | None = None) -> ChatTurnResponse: ...
    async def run_turn(self, turn_id: str) -> None: ...
    async def stream_turn_events(self, turn_id: str) -> AsyncIterator[ChatEvent]: ...
    async def cancel_turn(self, turn_id: str) -> ChatTurnResponse: ...
    async def retry_turn(self, turn_id: str) -> ChatTurnResponse: ...
    async def recover_turns(self) -> int: ...
```

职责限定：

```text
创建 turn 与初始 trace
统一 turn 状态机
统一调用 ContextGateway / Brain / Safety / ToolTask / ResponseComposer
统一事件落库与流式输出
统一取消、重试、恢复
统一收尾与 memory write 调度
```

禁止：

```text
不直接读写 HTTP 请求
不直接依赖 route handler
不绕过 Asset Broker / Capability Graph / Safety
不把 route-specific 文案长段写死在 runtime 中
```

### 3. ChatService

`ChatService` 保留为应用层 façade。

职责：

```text
组装 coordinator / repo / runtime 依赖
提供查询接口：conversation、turn、events、envelope、working state
提供与旧代码兼容的辅助方法
作为 registry 注入点
```

禁止：

```text
不再持有完整主链路编排
不再继续增长新的 route-specific 执行逻辑
不再直接成为所有运行态细节的宿主
```

### 4. TurnExecutionManager

`TurnExecutionManager` 继续承担运行调度基础设施：

```text
后台执行
并发控制
取消 token 管理
执行队列协调
恢复入口
```

但它不是业务 runtime。

## turn 状态机

统一 turn 生命周期：

```text
accepted
queued
running
completed
failed
cancelled
recovered
```

### 状态语义

#### accepted

```text
turn 已创建
user message 已落库
trace 已开始
尚未进入执行队列或后台执行
```

#### queued

```text
turn 已进入执行队列
等待被 runtime 拉起
```

#### running

```text
turn 正在执行
上下文、模型、工具、审批、收尾至少有一项正在进行
```

#### completed

```text
assistant 最终消息已写入
turn 事件闭合
trace 正常结束
memory write 已调度或显式跳过
```

#### failed

```text
runtime 在无法恢复的路径终止
必须有失败事件、错误码、recovery record
```

#### cancelled

```text
用户取消或上游取消
不得继续写出伪装完成的最终回复
```

#### recovered

```text
原 turn 因中断未完成
恢复流程完成了状态修复或重新挂起
不要求一定补出最终业务结果
```

## SSE 事件主序列

一个标准 direct turn 的最小事件序列固定为：

```text
turn.accepted
turn.queued            可选
turn.started
content.normalized     可选
context.started
context.ready
intent.detected
mode.selected
model.started          direct / model path
response.delta         流式可重复
model.completed        model path
response.completed
turn.completed
```

工具或审批路径允许插入：

```text
tool.requested
tool.started
tool.completed
approval.required
approval.resolved
task.handoff
recovery.attempted
```

统一原则：

```text
所有终态 turn 都必须有 turn.completed / turn.failed / turn.cancelled 三者之一
response.completed 只在用户可见回复已闭合时发出
任何工具、审批、恢复事件都必须归属到同一 trace_id
```

## 统一入口语义

### create_turn

负责：

```text
规范化输入
创建 turn / user_message / trace
建立 queue item
返回 turn_id、trace_id、stream_url
```

不负责：

```text
执行上下文构建
执行工具
生成最终回复
```

### run_turn

负责：

```text
拉起 turn
执行主状态机
写事件
写最终消息
收尾 trace
```

### cancel_turn

负责：

```text
发出 cancel token
终止后续可中断执行
把 turn 收敛为 cancelled
```

### retry_turn

负责：

```text
基于原 turn 生成 retry turn
保留 conversation 归属
建立 retry_of_turn_id 关联
```

### recover_turns

负责：

```text
扫描 running / queued 异常残留 turn
区分能继续执行、能安全补收尾、必须标记 recovered 的路径
```

## 从 chat.py 迁移出的能力分类

### 入口编排

迁移到 `ChatRuntime`：

```text
turn 生命周期推进
主分支调度
统一异常收束
统一 trace 结束
```

### 运行态

保留在独立中层或已有 coordinator：

```text
上下文构建
route 决策
模型调用
工具捷径
task handoff
response finalize
```

### 辅助协调器

继续保留：

```text
chat_experience
chat_quality_shadow
chat_continuation
chat_memory
chat_response
chat_privacy
```

### 查询接口

保留在 `ChatService`：

```text
get_conversation
list_conversations
get_turn
list_events
get_working_state
get_envelope
```

## route handler 与 channel gateway 约束

### route handler

`routes_chat.py` 只允许：

```text
解析请求
调用 SessionRuntime
返回 schema
统一错误处理
```

明确禁止：

```text
在 handler 中决定 direct / task / tool 分支
在 handler 中直接调用工具
在 handler 中改写可见回复
```

### channel gateway

微信、飞书等 gateway 只允许：

```text
完成渠道消息解析
会话路由
构建 ChatTurnRequest
调用 ChannelIngressRuntime / SessionRuntime
发送出站结果
```

明确禁止：

```text
在 gateway 内复制一份聊天主链路
在 gateway 内直接执行业务 route
在 gateway 内绕过 ChatRuntime 形成独立 turn 执行支路
```
