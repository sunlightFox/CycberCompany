# 开发计划目录

本目录用于承载最终态开发计划。开发只按一个产品版本推进：**最终封版版本**。阶段只是工程交付顺序，不代表产品存在 v1、v2、试验版或简化版。

## 开发原则

1. 每个阶段都服务最终产品，不做一次性 demo 代码。
2. 每个阶段文档只描述该阶段要完成的最终态能力切片。
3. 阶段之间允许逐步交付，但不允许引入未来必然推翻的临时架构。
4. 所有阶段必须遵守 `docs/README.md` 中的设计硬规则。
5. 聊天页、壳系统、资产中心、任务引擎、记忆系统、安全审计等核心边界从第一天就按最终态设计。
