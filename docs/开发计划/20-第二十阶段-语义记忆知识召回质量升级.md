# 第二十阶段：语义记忆与知识召回质量升级

## 摘要

第二十阶段聚焦“系统记得准、找得准、不过度注入”。当前 MemoryService 和 KnowledgeService 已经具备 source、supersede、向量检索、FTS fallback、selection_reason 和 trace；但向量提供者默认是本地 deterministic hash embedding，具备流程闭环，却不等同于高质量语义召回。

本阶段目标是把记忆和知识从“可用的本地向量流程”提升到“可配置的高质量语义检索体系”，同时保证敏感信息、过期记忆、冲突事实不会污染聊天上下文。

本阶段只做后端，不新增 UI。

## 阶段定位

第二十阶段回答：

```text
用户跨会话提到偏好时，系统是否能准确召回
旧记忆被更正后，是否不会继续污染回答
知识库内容是否能按语义而不是关键词命中
向量服务不可用时，是否能诚实 fallback
敏感记忆是否不会进入普通聊天上下文
每条记忆和知识命中是否有 source、selection_reason 和 trace
```

## 当前基线判断

| 能力 | 当前完成度 | 主要问题 |
|---|---:|---|
| 记忆写入 | 约 82% | source 和 supersede 已有，抽取质量偏规则 |
| 记忆检索 | 约 82% | local_hash_v1 可用，语义质量有限 |
| 知识索引 | 约 80% | chunk 和索引已有，复杂文档解析有限 |
| 检索解释 | 约 80% | selection_reason 已有，缺少冲突/时间解释 |
| 敏感边界 | 约 84% | 已做拦截和脱敏，需扩展 eval 覆盖 |

## 阶段原则

1. 记忆写入必须包含 source。
2. 向量召回不可用时才使用 FTS fallback，并记录 degraded_reason。
3. 新事实优先于旧事实，显式更正优先于隐式偏好。
4. 敏感记忆默认不进入普通聊天上下文。
5. Knowledge 和 Memory 检索都必须有 selection_reason。
6. embedding provider 可配置，但不能把密钥或原文敏感内容写入 trace。

## 阶段范围

### 本阶段必须完成

```text
EmbeddingProvider 抽象增强
local_hash / local_model / chroma provider 切换
Memory semantic ranking
Knowledge semantic ranking
rerank pipeline
temporal decay
supersede conflict resolver
sensitive memory gate
retrieval diagnostics
memory/knowledge eval
```

### 本阶段不做

```text
不做云端同步
不新增知识库前端
不把敏感原文发给未授权云端 embedding
不自动长期保存所有聊天
不让 Memory 修改任务状态
```

|---:|---|---|

## 小阶段 20.1：EmbeddingProvider 抽象增强

### 目标

让向量能力从固定 local_hash 提升为可配置 provider，同时保留本地无依赖 fallback。

### Provider 类型

```text
disabled
local_hash
local_model
chroma
external_compatible
```

### Provider 配置字段

```text
provider
embedding_model
embedding_dim
privacy_policy
max_batch_size
timeout_seconds
allow_cloud
secret_ref
status
degraded_reason
```

### 验收

```text
默认仍可本地运行
local_hash 作为 fallback 不被删除
高隐私文本不会发送到 cloud embedding
provider 状态进入 runtime contracts
embedding 调用写 trace 但不记录 secret
```

## 小阶段 20.2：Memory 召回排序升级

### 目标

提升长期记忆在聊天上下文中的命中准确度和稳定性。

### 召回层级

```text
pinned_memory
persona_memory
current_project_memory
semantic_vector
episodic_vector
recent_conversation_summary
fts_fallback
```

### 排序因子

```text
semantic_score
recency_score
source_reliability
explicitness
supersede_status
sensitivity_penalty
conversation_relevance
member_scope
```

### 验收

```text
更正后的新记忆优先旧记忆
过期记忆不会默认注入
敏感记忆除非明确需要且授权，否则不注入
每个 MemorySearchHit 有 retrieval_source 和 selection_reason
```

## 小阶段 20.3：Knowledge 索引与 chunk 增强

### 目标

提升知识库文档可检索性，避免 chunk 太粗、太碎或缺少来源。

### Chunk 字段

```text
knowledge_id
chunk_id
source_uri
source_type
title
section_path
page_number
char_start
char_end
token_count
content_hash
metadata
trace_id
```

### Chunk policy

```text
markdown_heading_aware
code_block_preserve
table_preserve_summary
sliding_overlap
max_token_limit
min_content_length
```

### 验收

```text
chunk 必须保留 source trace
重复内容按 content_hash 去重
知识索引失败不伪装成功
敏感路径仍被阻断
```

## 小阶段 20.4：Rerank 与冲突消解

### 目标

让召回结果不只是“相似”，还要“当前正确”。

### 冲突类型

```text
old_preference_vs_new_preference
project_rule_superseded
same_topic_conflicting_fact
low_source_reliability
sensitive_but_not_needed
knowledge_version_conflict
```

### 输出

```text
selected_items
suppressed_items
conflict_notes
selection_reason
confidence
requires_user_confirmation
```

### 验收

```text
旧偏好被 supersede 后不进入上下文
冲突事实触发澄清或保守回答
低置信度召回不会强行注入
suppressed reason 可查询
```

## 小阶段 20.5：敏感记忆与知识注入边界

### 目标

避免“记得越多越危险”，把敏感内容注入变成显式策略。

### 策略字段

```text
sensitivity_level
allowed_context_modes
requires_user_intent
requires_task_binding
requires_approval
redaction_policy
retention_policy
```

### 验收

```text
普通闲聊不注入 secret/account/wallet 类记忆
高风险动作需要的敏感信息只通过 Asset Broker handle
trace/audit 不出现敏感明文
知识库敏感命中默认只返回摘要或阻断原因
```

## 小阶段 20.6：检索诊断与评测

### 目标

用 eval 证明召回质量，而不是只证明接口可跑。

### 指标

```text
recall_at_k
precision_at_k
conflict_suppression_rate
supersede_accuracy
sensitive_injection_block_rate
fallback_correctness
latency_p95
```

### 必测 case

```text
稳定偏好召回
显式更正
跨会话项目上下文
知识库章节命中
向量不可用 FTS fallback
敏感记忆阻断
旧事实冲突澄清
```

## 阶段总验收标准

第二十阶段完成时必须满足：

```text
EmbeddingProvider 可配置并有 fallback
Memory/Knowledge 召回不仅可用，而且有质量指标
旧记忆污染新事实的问题被 supersede/rerank 控制
敏感记忆和敏感知识不会默认进入聊天上下文
所有召回结果都有 source、selection_reason、trace refs
eval 覆盖语义命中、冲突、fallback、敏感阻断和延迟
```
