# 第三十三阶段：重型压力测试缺口修复与聊天主链路硬化

## 摘要

第三十三阶段承接 `CHAT-E2E-20260430-POWER` 第 10 轮聊天主链路重型压力测试结果，把 108 条真实模型、任务、工具、Skill、MCP、浏览器和安全恢复场景暴露出的缺口整理成后续开发计划。

本阶段不是继续扩写测试，也不是修测试数据来让报告变绿，而是围绕 `docs/测试/聊天主链路/2026-04-30/` 产出的真实失败证据，按安全优先、主链路优先、能力补齐、稳定性硬化的顺序推进后端修复。

本阶段继续遵守仓库后端优先约束：

```text
不新增前端页面
不新增 React / Tauri / 桌面窗口
不绕过 API、Service、Repository 分层
不绕过 Safety、Approval、Trace、Asset Broker、Capability Graph
不直接清理当前 data 来掩盖问题
不把未实现能力包装成已完成
不把真实模型失败用假回复替代
```

第三十三阶段的核心目标是让第 10 轮重型测试从：

```text
PASS 76 / FAIL 32 / BLOCKED 0
待修复问题 46 个：P0 10、P1 18、P2 18
```

收敛到：

```text
P0 全部清零
P1 主链路问题全部清零或转成有证据的 accepted risk
P2 浏览器、Skill/MCP、任务证据、格式质量问题有明确修复或阶段边界
108 条重型用例可稳定重跑并生成无明文敏感信息的报告
```

## 当前基线判断

| 能力 | 当前状态 | 第三十三阶段要收敛的缺口 |
|---|---|---|
| 真实模型聊天 | 预检通过，基础聊天可用 | 长输出、严格 JSON、隐私高输入下仍会失败或误路由 |
| 知识总结 | 多数可用 | 部分总结请求被识别成任务，导致 `model.completed` 缺失或任务误创建 |
| 记忆 | 敏感记忆可拒写 | 脱敏断言、敏感证据、临时记忆边界和 memory source 仍需收敛 |
| Persona/Heart | 基础边界可回答 | 拒绝类回复被误判泄漏，边界表达和测试断言都需区分 |
| 任务和工具 | 基础执行可用 | database lock、task replay、trace 完整性、错误状态语义需硬化 |
| Skill | 安装/启用/匹配/运行存在异常 | 测试专用 Skill 生命周期、重复安装、状态转换和错误语义不稳 |
| MCP | 注册基础可用 | sync 502、工具调用 403、resource/prompt/tool 隔离和测试 transport 需补强 |
| 浏览器执行 | open/snapshot/screenshot/download 部分可用 | 登录成功/失败证据、慢页面 timeout、404 语义、file URL、交互工具参数契约需要收敛 |
| 安全恢复 | 部分安全拒绝可用 | 隐私高输入、越权、钓鱼、威胁、prompt injection 在锁竞争时出现完成性失败 |
| 测试平台 | 可生成 07/08/09 报告 | 脱敏函数误报、内部字段误判、并发执行互斥和 SQLite 锁治理需纳入开发 |

## 阶段原则

1. P0 优先，所有 secret、token、password、cookie、private key、本地敏感路径不得出现在回复、事件、trace、task replay、artifact、报告中。
2. 区分产品缺陷和测试断言缺陷。拒绝泄漏“系统提示/trace”的安全回复不应被误判为泄漏。
3. 知识总结、解释、压缩、表格、严格 JSON 默认走直接模型回复，不能因为出现“工具”“Skill”“浏览器”等词就误建任务。
4. 用户明确说“不执行”“不要创建任务”“不要调用工具”时，最新指令优先。
5. 浏览器、终端、文件、MCP、Skill 执行必须绑定任务、审批、trace 和 replay 证据。
6. 错误要可诊断。404、timeout、403、422、502、database lock 需要稳定错误模型和恢复建议。
7. 测试执行必须可重复。重型 E2E、pytest、旧批次脚本不能互相抢当前 `data`。
8. 修复后必须用真实模型重跑对应 case，不允许改低断言或使用假回复代替。
