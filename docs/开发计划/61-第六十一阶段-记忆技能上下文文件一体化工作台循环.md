# 第六十一阶段 - 记忆、Skill 与上下文文件一体化工作台循环

## 阶段背景

Hermes 之所以“像一个会成长的 agent”，关键不是只会聊天，而是它会把记忆、技能、上下文文件和会话状态拼成一个持续工作的工作台。当前项目已经有 Memory、Skill、Context Gateway、Shell Runtime 和知识检索能力，但这些能力还偏服务化分散，离“自动加载、自动沉淀、自动校正”的统一工作台循环还有距离。

这一阶段的目标，是把 memory / skills / context files 变成一个可以持续迭代的 agent 工作台，而不是散落在多个 API 和后台任务里的能力拼盘。

本阶段继续只做后端、schema、migration、repository、service、API、tests、evals 和文档；不新增前端页面、组件、样式、Tauri 窗口或桌面端交互代码。

## 参考结论

### OpenClaw 采用点

OpenClaw 的 workspace 思路强调持续上下文：

```text
工作区里能看见当前任务、历史痕迹和可复用内容
能力不是一次性调用，而是可回看、可续跑
```

本项目采用：

```text
围绕成员、任务和组织，生成可版本化的上下文包
记忆和 Skill 都要能回填到后续 turn
```

### Hermes Agent 采用点

Hermes 的优势在于把记忆和技能变成默认工作状态：

```text
会自动带入稳定偏好
会把可复用能力加载到会话里
会在结束后再把经验写回
```

本项目采用：

```text
会话启动时自动加载必要记忆、可用 Skill 和上下文文件
会话结束后自动沉淀候选记忆、Skill 候选和工作台摘要
```

## 核心目标

本阶段完成后，后端应支持：

```text
成员级稳定记忆快照
Skill 候选与已安装 Skill 的统一匹配与回填
上下文文件的生成、版本化、注入和回写
会话启动、执行、反思、沉淀的闭环工作台循环
```

## 阶段原则

1. 记忆不是孤立表，而是可被上下文网关持续消费的工作材料。
2. Skill 不是只挂在工具上的标签，而是能进入工作台循环的做事方法。
3. 上下文文件不是手写文档，而是系统生成并可回放的状态快照。
4. 所有写回都必须带 source、trace 和可解释原因。
5. 不把未验证的候选内容直接升级成稳定事实。

## 阶段范围

### 本阶段必须完成

```text
memory / skill / context pack 的统一构建器
工作台启动时的自动加载策略
执行后记忆抽取、Skill 候选生成和上下文回写
上下文文件的版本管理、差异查看和回放
```

### 本阶段不做

```text
不让 Skill 绕过 Asset Broker 或 Capability Graph
不把上下文文件伪装成真实数据库主数据
不把候选记忆直接写成最终记忆而不经过审查
不新增 UI
```

## 主要待补

```text
工作台级上下文包
记忆与 Skill 的双向沉淀
上下文文件的版本化与回写
会话结束后的反思循环
```

## 验收标准

```text
会话开始时能自动装载稳定记忆和可用 Skill
会话结束后能自动产出候选记忆和上下文文件更新
同一成员的后续会话能复用此前沉淀
所有沉淀都能追溯 source 和 trace
```
