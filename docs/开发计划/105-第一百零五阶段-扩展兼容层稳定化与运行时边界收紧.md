# 第一百零五阶段 - 扩展兼容层稳定化与运行时边界收紧

## 阶段定位

第一百零一阶段把扩展兼容层、渠道平台插件化和 OpenClaw/Hermes 相关链路铺到了可用状态，但现在卡住继续推进的，不是“有没有扩展能力”，而是扩展运行时边界还不够紧，兼容层还会把不该启动的东西误拉起来。

本阶段的目标不是继续扩能力，而是把 phase101 暴露出来的兼容性和运行时分层问题收口到可解释、可诊断、可验证的稳定状态。

## 目标

```text
收口 phase101 暴露的 OpenClaw / Hermes 扩展兼容问题
让扩展导入、预览、安装、诊断、启用链路稳定
明确 external runtime、bridge plugin、plan-run 的职责边界
把 runtime diagnostics / next_actions 的契约统一下来
```

## 重点

### 105.1 导入边界收紧

```text
扩展导入不应误启动外部 MCP 进程
本地目录扫描、manifest 解析、兼容性预检查应保持纯静态行为
只有显式启用或明确执行 runtime 相关动作时才允许进入运行态
```

### 105.2 诊断契约统一

```text
runtime diagnostics 的字段命名与层级统一
next_actions 只给出当前层真实可执行的后续动作
兼容层、桥接层、真实 runtime 不再混出一份模糊状态
```

### 105.3 运行时分层

```text
external runtime 负责真实运行与健康信息
bridge plugin 负责接线与兼容适配
plan-run 负责计划执行与预览，不偷带运行时副作用
```
