# 第三十七阶段 - 持久浏览器会话与网页登录资产化

## 阶段背景

当前后端已经有 `browser.open/search/snapshot/fill/type/click/submit/screenshot/download` 等工具，也能通过账号资产和 Asset Broker 执行受控登录、发布草稿和审批。但这些能力仍偏“一次性工具调用”：

```text
浏览器上下文不稳定
登录态没有成为可授权、可撤销、可审计的资产
cookie/session/profile 没有明确生命周期和敏感级别
网页自动化证据更多是 snapshot/screenshot，不等于真实持久浏览器工作流
```

对标 OpenClaw / Hermes Agent，用户最直观的期待是：“它能打开我常用的网站，用我授权的会话帮我做事，但不要泄露账号、cookie 或越权乱点。”第三十七阶段要把浏览器会话、网页登录态、profile 和网站权限纳入后端稳定契约，让浏览器自动化从“临时打开页面”升级为“受控的浏览器会话资产”。

本阶段仍严格后端优先，只做 schema、migration、repository、service、API、tests、evals 和文档，不新增 UI、Tauri、React、样式或桌面端交互代码。

## 核心目标

本阶段完成后，后端应能支持：

```text
创建受控 BrowserProfile 记录
将网页登录态作为 account/browser_session 类资产管理
通过 Asset Broker 发放短期 browser session handle
浏览器工具按 profile/session handle 执行，不直接接触明文 cookie
登录、下载、上传、提交、跨站带凭据动作经过 Safety 和 Approval
浏览器 profile 可暂停、撤销、清理、过期和审计
每次网页动作都有 snapshot/screenshot/network/console/artifact/trace 证据
网页内容默认标记 untrusted，不可覆盖系统指令
```

## 阶段原则

1. 浏览器登录态是敏感资产，不是普通工具参数。
2. 模型和 Skill 不得读取明文 cookie、token、localStorage、浏览器 profile 路径。
3. 浏览器工具只能通过 Asset Broker 解析短期会话句柄。
4. 真实浏览器 profile 与临时任务 profile 必须区分。
5. 访问私网、metadata、file URL、本机敏感路径必须 hard block 或高安全处理。
6. 上传、提交、发布、付款、改账号设置等动作必须审批。
7. 页面内容、MCP resource、下载文件、网页脚本输出都视为不可信外部内容。
8. 下载文件进入 artifact quarantine，不自动打开、不自动执行。
9. 不把“打开过页面”说成“完成了登录/提交/下载”；必须以工具事件和 artifact 为证据。
10. 不新增 UI；未来 UI 只消费本阶段 API 和状态。

## 对标结论

### OpenClaw 采用点

```text
用户更关心浏览器会话连续性，而不只是单次打开网页
登录态、cookie、下载目录、任务证据需要在 agent 和浏览器之间稳定传递
浏览器能力要能服务调研、账号操作、文件下载、网页整理等复杂任务
```

对我们的启发：

```text
浏览器 profile 必须资产化
browser tools 必须支持 profile/session handle
下载、截图、network、console 都要进入统一 evidence 模型
```

### Hermes Agent 采用点

```text
浏览器工具要结合 URL safety、hardline block 和审批
危险 URL、私网地址、metadata、file URL、同形混淆等不能靠用户确认绕过
页面内容可能包含 prompt injection，必须隔离为 untrusted context
```

对我们的启发：

```text
新增 BrowserSafetyPolicy
把 URL 解析、目标分类、profile 敏感度和 action risk 统一到 ActionRiskDecision
```

## 关键实现要求

### Session handle

```text
Asset Broker query 返回 browser_session handle 摘要
resolve_for_tool 只返回最小 profile/session 引用，不返回 cookie 明文
真正 cookie/localStorage 只在浏览器执行边界内解密或加载
handle 绑定 member、task、conversation、action、expires_at
profile 被 revoke 后所有 active handle 失效
```

### URL safety

```text
解析 URL scheme、hostname、port、ip、redirect chain
阻断 file://、data: 可执行载荷、javascript:、metadata IP、localhost 敏感端口
私网地址默认 deny，除非本地测试策略明确允许且无敏感路径
URL 中出现 token/password/cookie/private_key 等敏感 query 时默认脱敏并升级风险
homograph 或 punycode 异常进入 review/deny
```

### Evidence bundle

```text
每次浏览器工具调用都写 browser_evidence
snapshot 内容脱敏并标记 untrusted
screenshot/download 写 artifact，带 checksum、content_type、size
network 只保存元数据，不保存 Authorization/Cookie 明文
console 只保存 error/warning 摘要并脱敏
```

### 审批策略

```text
只读公开网页 snapshot 可 allow/inform
持久 profile 登录、下载、上传、提交、发布、修改账号设置必须 approval
跨域带凭据行为升级风险
下载可执行文件、压缩包、脚本、未知 MIME 升级风险并进入 quarantine
```

## 必测用例

```text
创建 browser profile，默认策略正确
撤销 profile 后 session handle 无效
Asset Broker 查询 browser_session 返回脱敏 handle
browser.snapshot 使用 session_handle_id 不泄露 cookie
browser.download 生成 download artifact 和 browser_evidence
browser.screenshot 生成 screenshot artifact 和 browser_evidence
metadata URL / file URL / javascript URL 被阻断
私网 URL 在无测试 allow policy 时被阻断
登录态资产 secret 不出现在 API、trace、audit、diagnostic 中
任务 replay 能列出浏览器证据
```
