# 第二十九阶段：长期体验评测 CI 化与封版规模化验证

## 摘要

第二十九阶段聚焦“最终封版的规模化证明”。当前第二十三阶段已经把本地 ruff、mypy、pytest、eval evidence、accepted risk、diagnostic bundle 和 release report 串成闭环；但 runtime contracts 仍明确 `ReleaseGate` 为 degraded，`PersonaHeartLongitudinalEval` 是 local deterministic evidence，`VerificationClosure` 也声明 local verification 不能替代 CI、多平台和长周期负载。

本阶段目标是把本地封版证据扩展为可持续、可复跑、可诊断的 CI 化验证体系，覆盖长对话、长任务、模型辅助能力、MCP/工具安全、性能、降级和多平台基础兼容。

本阶段只做后端验证、评测、诊断、脚本和报告，不新增 UI。

## 阶段定位

第二十九阶段回答：

```text
本地通过是否能被 CI 稳定复现
长对话和长任务是否能在预算内完成
模型辅助能力是否有质量回归监控
accepted risk 是否有复查条件和过期机制
性能、内存、数据库迁移、备份恢复是否可持续验证
ReleaseGate 是否能从 local smoke 升级为 release-grade evidence
```

## 当前基线判断

| 能力 | 当前状态 | 缺口 |
|---|---|---|
| ruff/mypy/pytest | 本地通过 | 需要 CI 化和耗时分层 |
| EvalEvidenceAggregator | implemented | 需扩展阶段 24-28 |
| AcceptedRiskRegistry | implemented | 需过期、复查、owner 机制 |
| PersonaHeartLongitudinalEval | implemented | local deterministic，缺长周期/多样化 |
| ReleaseGate | degraded | 本地 deep smoke，缺 CI/规模化 evidence |

## 阶段原则

1. 不降低测试断言换取通过。
2. CI 和本地命令必须一致或有明确差异。
3. 安全、secret 泄漏、越权、审批绕过为零容忍。
4. accepted risk 必须有 recheck_trigger 和 expiry。
5. 长耗时测试必须分层运行，不能阻塞日常快速回归。
6. ReleaseGate 必须引用真实 evidence，而不是手写状态。

## 阶段范围

```text

long-run eval suites
performance benchmark

backup/restore validation

release-grade report
```

### 本阶段不做

```text

不做云端 SaaS
不替代真实用户体验研究
不把 CI 失败标成 accepted risk
不隐藏 flaky 测试
```

## 小阶段 29.1：CI 命令矩阵

### 目标

明确本地和 CI 的验证层级。

### 命令层级

```text
fast:
ruff
mypy
unit tests

api:
apps/local-api/tests -m "not slow"

security:
secret scanner
permission boundary validator
tool/mcp/sandbox eval

release:
all tests
all evals
release gate
diagnostic bundle
```

### 验收

```text
pyproject markers 完整
CI 脚本可本地运行
失败输出包含阶段和 suite
```

## 小阶段 29.2：长对话与长任务 eval

### 目标

验证系统在更长时间尺度下不漂移、不泄漏、不越权。

### Long eval

```text
50-turn dialogue continuity
multi-session memory recall
preference correction drift
long agent budget management
tool failure recovery chain
persona consistency drift
MCP untrusted content persistence
```

### 指标

```text
continuity_score
memory_drift_count
unsafe_action_count
internal_leakage_count
budget_violation_count
latency_p95
trace_completeness
```

### 验收

```text
安全违规为 0
secret 泄漏为 0
预算违规为 0
长对话 continuity 达阈值
```

## 小阶段 29.3：性能与资源基准

### 目标

确保主链路、记忆检索、任务执行和 ReleaseGate 在可接受资源范围内。

### 基准

```text
chat turn latency p50/p95
brain decision latency
context gateway latency
memory search latency
knowledge search latency
tool runtime overhead
release gate duration
trace storage growth
diagnostic bundle size
```

### 验收

```text
性能报告进入 release report
p95 超阈值标记 degraded 或 blocker
trace/diagnostic 大小受控
```

## 小阶段 29.4：迁移与备份恢复验证

### 目标

证明 schema/migration/repository/service/API 的最终态可安装、可升级、可恢复。

### 验证

```text
fresh database migration
upgrade from previous snapshot
seed idempotency
backup create
backup restore
restore integrity check
runtime contracts after restore
```

### 验收

```text
fresh install 通过
upgrade install 通过
backup/restore 后 eval smoke 通过
restore 不泄漏 secret
```

## 小阶段 29.5：Accepted risk 生命周期

### 目标

避免 accepted risk 永久沉淀成无人处理的风险。

### 字段

```text
risk_id
module
why_accepted
scope
mitigation
owner_phase
created_at
expires_at
recheck_trigger
promotion_rule
```

### Promotion rule

```text
expired -> blocker
new exploit/eval failure -> blocker
scope expanded -> review_required
mitigation missing -> blocker
```

### 验收

```text
release report 标出即将过期风险
过期风险阻断 go
每个 degraded contract 有 risk owner
```

## 小阶段 29.6：ReleaseGate 升级

### 目标

把 ReleaseGate 从 local deep smoke 升级为 release-grade evidence gate。

### Gate 输入

```text
ruff status
mypy status
pytest status
eval status
security scanner status
secret scanner status
permission validator status
trace integrity status
backup restore status
performance benchmark
accepted risk status
diagnostic bundle
```

### Go 条件

```text
zero security blocker
zero secret leakage
zero permission bypass
all required eval passed
ruff/mypy passed
pytest required suites passed
accepted risks unexpired
performance within thresholds or explicitly degraded
```

### 验收命令

```text
.venv\Scripts\python.exe -m pytest apps\local-api\tests\test_phase29_release_scale_verification.py
.venv\Scripts\python.exe -m ruff check .
.venv\Scripts\python.exe -m mypy .
.venv\Scripts\python.exe -m pytest
```
