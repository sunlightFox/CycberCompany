## 测试分层

### 1. 单元测试

必须覆盖：

```text
runtime
context
routing
composer
memory
safety
hook governance
```

#### runtime

```text
turn 状态推进
事件序列
取消 / 重试 / 恢复
```

#### context

```text
ContextPacket 分层
token budget 裁剪
session / presence / memory 优先级
untrusted_context 标注
```

#### routing

```text
direct vs tool loop vs task handoff
clarification 触发
latest instruction override
```

#### composer

```text
pending honesty
visible filter
不同场景可见回复
渠道差异化渲染
```

#### memory

```text
source 强制
memory candidate 抑制
supersede / archive
```

#### safety

```text
privacy classification
approval required
deny
tool risk classification
```

### 2. 集成测试

范围固定为：

```text
/api/chat/turn
  -> turn accepted
  -> runtime execution
  -> event stream
  -> final state
```

必须覆盖：

```text
direct turn
tool loop turn
task handoff turn
pending approval turn
failure turn
retry turn
recover turn
```

### 3. 渠道回归

至少覆盖：

```text
wechat inbound -> session route -> turn -> outbound
feishu inbound -> session route -> turn -> outbound
```

关注点：

```text
会话续接正确
不串 conversation
重复消息不重复执行
pending action 绑定唯一
```

### 4. 恢复测试

必须覆盖：

```text
模型失败
context build 失败
tool timeout
running turn 中断
cancelled turn
retry after failed
recover incomplete turns
```

### 5. 安全测试

必须覆盖：

```text
approval required
deny path
secret redaction
trace redaction
response visible redaction
untrusted external content 标注
跨渠道未绑定 fail closed
```

### 6. 质量测试

必须覆盖：

```text
聊天真实性
多轮连续性
当前消息优先
pending action honesty
失败诚实性
边界回复自然度
```

## 关键用例矩阵

### 本地普通聊天

预期：

```text
direct reply 成功
trace 完整
不创建多余 task
不泄漏内部信息
```

### 微信 / 飞书入站

预期：

```text
正确续接 session
不串 conversation
重复入站不重复执行
```

### 有 pending action

预期：

```text
回复不伪装已完成
确认、拒绝、修改只绑定唯一 pending action
```

### 高风险工具

预期：

```text
必须进入 approval.required 或 deny
未批准不得执行真实动作
```

### 只读工具

预期：

```text
允许单轮完成
结果可回注
最终回复自然、诚实
```

### 模型失败

预期：

```text
turn 进入 failed 或 recovered 的预期路径
用户看到诚实失败说明
```

### context build 失败

预期：

```text
有失败事件
有 recovery 记录
无伪装完成回复
```

### retry / cancel / recover

预期：

```text
事件序列一致
状态闭合
关联键正确
```

### 记忆写入

预期：

```text
必须有 source
临时态不误写长期记忆
```

### redaction

预期：

```text
trace / audit / response 不泄漏 secret、token、cookie、内部诊断原文
```

## 发布 gate

后端提交前必须跑：

```text
pytest
ruff check .
mypy .
```

其中 pytest 必跑集至少包括：

```text
聊天 turn 主链路
渠道回归
approval / deny
memory source
trace / response redaction
recover / retry / cancel
```

### 阶段完成红线

只要命中以下任一情况，本阶段视为未通过：

```text
未执行说成已完成
pending action 误释放
跨渠道串 conversation
高风险动作绕过 approval
response 泄漏 trace_id 或 secret
memory write 无 source
```

## 证据要求

每个主阶段合并前至少提供：

```text
关键 pytest 结果
代表性 trace / event 样本
渠道回归结果
失败与恢复样本
redaction 样本
```

## 验收标准

```text
聊天主链路测试矩阵按能力分层，而不是按偶发 bug 补丁
发布 gate 与质量红线清晰
任何后续主链路重构都可以映射到明确测试集
```
