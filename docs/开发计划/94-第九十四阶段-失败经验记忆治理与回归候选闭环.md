# 第九十四阶段 - 失败经验记忆治理与回归候选闭环

## 目标

为聊天主链路补齐“失败经验”治理层：把运行失败、工具执行失败与人工复核后的失败经验纳入可检索记录，并在重复出现时沉淀为回归候选。

## 范围

- 新增 `failure_experience_records` 与 `regression_candidates` 表。
- 新增 `FailureExperienceService`，负责记录失败、人工审核、回归候选生成、advisory recall。
- 失败经验只允许写入 `failure_advisory` 记忆，不直接改写 durable user fact。
- 高风险或低证据失败默认进入 `pending_review`。
- 相同 `recurrence_key` 在 7 天内重复两次及以上，生成或更新 regression candidate。
- `ContextGateway` 可召回已批准的 failure advisory，并写入 `context_diagnostics.failure_advisories`。
- `release` 与 `chat-mainline-readiness` 暴露 phase94 状态、审核动作和回归阈值。

## API

- `GET /api/memory/failure-experiences`
- `POST /api/memory/failure-experiences/{failure_id}/review`
- `GET /api/memory/regression-candidates`

## 验收

- 失败记录可查询、可审核、可 tombstone。
- 审核批准后可生成 advisory memory。
- 重复失败可自动形成 regression candidate。
- readiness、release report、runtime contract 均可见 `phase94.failure_experience_governance.v1`。
