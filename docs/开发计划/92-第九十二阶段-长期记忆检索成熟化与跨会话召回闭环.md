# 第九十二阶段 - 长期记忆检索成熟化与跨会话召回闭环

## 阶段定位

当前系统在“记忆写入必须有 source”这件事上已经比较稳，  
但距离高质量长期记忆系统还有明显差距：能记，不等于记得好、找得到、纠得动、压得住。

第九十二阶段的目标是把记忆从“合规可写入”推进到“用户真正感知到有帮助的可检索长期记忆”。

## 阶段目标

```text
建立跨会话可控检索能力
把偏好、事实、经验、任务状态型记忆分层治理
建立记忆纠错、覆盖、压缩与过期策略
让 ResponseComposer 与 ContextGateway 能稳定使用记忆而不是偶发碰运气
```

## 核心问题

当前记忆系统仍存在几类成熟度缺口：

```text
跨会话召回能力弱，很多记忆只在当前 session 里显得有效
记忆类型混杂，偏好、事实、临时状态、一次性 TODO 容易混层
纠正过的记忆能不能压过旧记忆，还缺稳定机制
记忆召回结果缺少质量度量，容易“写了很多，但答不出来”
```

## 本阶段范围

### 必须完成

```text
增加跨会话检索入口与 scoped recall contract
细化 preference / fact / experience / transient working state 的记忆分类
建立 correction / supersede / expiry / compression 规则
补记忆召回质量评测与真实聊天回归
```

### 明确不做

```text
不做完全自动长期人格进化
不把所有临时会话状态都强写入长期记忆
不绕过 source / trace / safety 约束
```

## 实施拆解

### 92.1 记忆类型分层

目标：

```text
让系统明确知道“这是一条偏好、这是一条事实、这是一条经验、这是一条临时状态”
```

交付：

```text
memory classification contract
memory source + memory scope + memory durability 字段补强
working state 与 long-term memory 的边界规则
```

### 92.2 跨会话检索入口

目标：

```text
让 ContextGateway 能基于 conversation 之外的历史证据安全召回相关记忆
```

交付：

```text
cross-session recall API
scoped search policy
result ranking / freshness / confidence signals
```

### 92.3 纠错、覆盖与压缩

目标：

```text
让“最新更正”稳定压过旧记忆，同时防止长期堆积失控
```

交付：

```text
superseded memory relation
preference correction flow
compression / summary compaction policy
expiry and stale-memory handling
```

### 92.4 记忆质量评测

目标：

```text
把记忆系统从“有没有写入”提升到“有没有帮助回答”
```

交付：

```text
memory recall precision / correction accuracy / stale recall rate 指标
跨会话偏好召回 regression
真实聊天记忆场景回归
```

```text
系统具备跨会话长期记忆召回能力
```
