## 最终产品一句话

这是一个可单机部署的个人智能体操作系统：前台是极简聊天，后台是组织化、多成员、多资产、多技能、多工具、长期记忆和可审计任务执行系统。首发只做公司壳，但底层从第一天就按可换壳架构设计。

## 设计硬规则

1. 聊天是唯一主入口，但管理页面必须完整存在。
2. 聊天界面只显示当前聊天对象的人名、头像、状态、消息和必要确认按钮，不显示组织、壳、部门树或资产归属。
3. 首发公司壳 MVP，但底层不能写死公司、员工、老板等概念。
4. 壳只改变 UI 标签、菜单名称、模板和引导文案，不自动改底层数据值。
5. 资产管理一级菜单可以随壳改名，但二级分类固定为大脑、账号、钱包、硬件、知识库。
6. 系统管理一级菜单和所有二级菜单固定，不随壳改名或改结构。
7. 智能体默认独立，不天然感知部门、其他成员和资产中心。组织资源必须通过 Resource API、Tool、MCP、知识库检索和 Skill 受控注入。
8. Skill 负责“怎么使用资源完成任务”，不承担全部资源发现职责。
9. 默认上手舒适，创建智能体只需要配置大脑、名字头像和人格模板；复杂权限不在首屏打扰用户。
10. 高风险动作必须确认、审计、可回放。
