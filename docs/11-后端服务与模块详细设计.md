## 后端目标

后端是本地智能体操作系统的内核。它负责：


后端必须可单机运行，默认不依赖云服务。云模型只是可选能力。

## 依赖方向

允许：

```text
api -> application services
chat-runtime -> context-gateway / brain / task-engine / response-composer / memory / trace
task-engine -> skill-engine / asset-broker / safety / trace
skill-engine -> asset-broker / tools / mcp registry
asset-broker -> capability-graph / asset repo / secret store
safety -> capability-graph / policy config / trace
context-gateway -> memory / capability-graph / shell-runtime
```

禁止：

```text
工具直接访问 secret
模型路由直接读数据库密钥
Skill 绕过 Asset Broker
API handler 直接执行工具
UI 参数直接进入 shell command
Memory 直接改任务状态
壳系统直接改成员字段值
```

## 应用启动

`lifespan.py` 启动顺序：

1. 加载配置。
2. 初始化日志。
3. 打开 SQLite。
4. 执行 migration。
5. 初始化 repositories。
6. 加载 shell 配置。
7. 确保默认组织和公司壳存在。
8. 初始化向量库。
9. 加载 Skill registry。
10. 加载 MCP registry，但按需启动服务。
11. 初始化模型 adapter registry。
12. 启动后台 worker。

健康检查：

```http
GET /health
```

返回：

```json
{
  "status": "ok",
  "db": "ok",
  "vector_store": "ok",
  "default_shell": "company",
  "version": "0.1.0"
}
```

### Token Budget

上下文预算：

| 部分 | 默认占比 |
|---|---:|
| 系统和开发指令 | 10% |
| 当前用户输入 | 10% |
| 会话摘要 | 15% |
| 长期记忆 | 20% |
| 能力和资源摘要 | 15% |
| 工具结果 | 20% |
| 预留 | 10% |

超预算处理：

```text
优先保留当前用户输入
保留高置信长期偏好
压缩旧工具结果
丢弃低相关 episodic memory
只保留资源句柄摘要
```

## 后台 Worker

### Task Worker

负责长任务继续执行：

```text
从任务队列拉取 runnable task
恢复 task state
执行下一步
遇到 approval 停止
失败重试
写 trace
```

### Memory Worker

负责异步记忆处理：

```text
抽取候选
评分
冲突检测
向量写入
写入 source trace
```

## API Handler 规范

每个 handler 只做：

```text
解析请求
调用 service
返回 schema
处理统一错误
```

不做：

```text
复杂业务逻辑
工具执行
prompt 拼接
权限绕过
直接操作多个 repository
```

## 测试要求

| 模块 | 必测 |
|---|---|
| Chat Runtime | direct、task、失败、trace |
| Context Gateway | token budget、记忆选择、资源脱敏 |
| Brain | 意图识别、模式选择、隐私路由 |
| Asset Broker | 句柄脱敏、过期、权限拒绝 |
| Capability Graph | allow、deny、approval reason |
| Safety | R0-R7、DLP、注入 |
| Tool Runtime | schema 校验、trace、approval |
| Shell Runtime | 菜单映射、系统管理固定、切壳不改数据 |
