# 聊天链路阶段 1 兼容层清单

## 当前保留在 `chat.py` 的兼容入口

- `ChatService._presence_runtime_inputs(...)`
- `ChatService._response_plan_for_status(...)`
- `ChatService._response_plan_for_action_status(...)`
- `_handle_host_filesystem_list(...)` 等直达路由方法的薄代理
- turn finalize / fail / event persist 主干

这些入口当前允许继续保留，因为它们还承担：

- facade 注入边界
- runtime 组装
- 向旧调用路径提供兼容壳

## 本轮已迁出的直达分支

- host filesystem readonly
- browser read page readonly
- browser search readonly
- desktop capability boundary
- terminal readonly command
- office document direct route

以上行为主体已下沉到 `apps/local-api/app/services/chat_direct_routes_runtime.py`。

## 第一阶段末可删除的纯转发壳

- `chat.py` 中仅转发到 `ChatDirectRoutesRuntime` 的直达路由方法
- 不再承担额外判断的 response-plan 兼容包装壳

## 删除条件

- 所有 visible-text 入口都经由 facade helper 接入 `presence_runtime`
- 迁出分支完成至少两轮主链路回归
- structured payload / trace / approval / task 边界未退化
