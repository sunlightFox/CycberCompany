# 聊天话术重构清单

## 参考结论

本轮只复用 OpenClaw 与 Hermes 的结构思想，不逐字搬运长文案。

- OpenClaw 侧重点：系统提示分层、当前消息与历史分离、进度草稿、静默回复、渠道投递边界。
- Hermes 侧重点：SOUL 身份层、记忆与 session continuity、技能预处理、稳定 prompt 与每轮临时上下文分离。

## 现有硬编码出口

- 模型提示词：`apps/local-api/app/services/chat_model.py`
- 微信续跑重写提示词：`apps/local-api/app/services/chat_continuation.py`
- 用户可见话术目录：`services/response-composer/response_composer/opening_copy.py`
- ResponsePlan 文案和失败回退：`services/response-composer/response_composer/contracts.py`
- 自然语言审批确认：`apps/local-api/app/services/natural_chat.py`
- 质量边界回复：`apps/local-api/app/services/chat_quality.py`
- 主链路补丁型边界文案：`apps/local-api/app/services/chat.py`

## 统一场景 Taxonomy

当前统一话术层要求覆盖：

- `casual_chat`
- `knowledge_answer`
- `clarification`
- `task_status`
- `approval_required`
- `action_status`
- `tool_boundary`
- `memory`
- `skill`
- `asset`
- `privacy`
- `professional_advice`
- `multimodal`
- `channel_silent`
- `progress_draft`
- `failure_recovery`
- `notification`

## 退出清单

- 新模型提示词只进入 `response_composer.chat_voice.ChatPromptAssembler`。
- 新用户可见短话术只进入 `opening_copy.py` 的 catalog。
- 新场景必须先补 `chat_voice.py` 的 taxonomy 与 metadata。
- ResponsePlan 必须携带 `voice_policy_version`、`scenario_id`、`channel_profile`、`delivery_mode`。
- 模型调用 trace metadata 必须携带 prompt assembly 版本与 prompt snapshot。
- 用户可见回复不得泄露 `trace_id`、`approval_id`、`tool_call_id`、`model_safe_text` 等内部词。
- 没有经过 Task/Tool/Safety/Approval 的真实动作，不得写成已经执行或已经完成。
