## 发现的问题和修复

第一次真实链路测试没有执行安装，原因是“豆包”没有解析到高置信官方安装源，计划状态为 `manual_only`。证据保存在 `00-summary.json`、`01-plan.json`。

本次实测后已进一步调整：

- 去掉 `豆包` / `doubao` 到 `ByteDance.Doubao` 的专用官方来源 hint。
- 改为允许模型只补充候选包 ID、别名、官网和厂商域名；后端仍必须用官方 winget manifest、可信官网、SHA256 等确定性校验后才进入审批安装。
- manifest 查询保留可见包 ID 的原始大小写，避免 GitHub winget manifest 路径大小写敏感导致 `ByteDance.Doubao` 查询失败。
- 增加测试覆盖中文名通过模型候选解析到官方 manifest，以及 manifest 大小写保留。

注意：上面的真实安装证据来自本次成功安装运行；当时后端使用了官方来源 hint。当前代码已按后续要求移除豆包专用 hint，后续同类未知软件需要有可路由模型返回候选，且候选通过确定性校验后才会安装；没有模型或校验失败时仍会安全降级为 `manual_only`。
