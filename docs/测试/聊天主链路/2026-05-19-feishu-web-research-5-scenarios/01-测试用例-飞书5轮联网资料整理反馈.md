# 01 测试用例 - 飞书 5 轮联网资料整理反馈

目标：验证飞书渠道作为入口时，聊天主链路是否能稳定完成“收到用户问题 -> 触发浏览器搜索 -> 收集网页线索 -> 整理成面向用户的反馈”这一类资料型场景。

关注点：
- 入口正确：消息从飞书进入，不绕过聊天主链路
- 路由正确：命中 `browser_search_with_citation`
- 边界正确：只做只读搜索，不创建任务，不伪造已下载/已提交/已预约
- 证据正确：回复里明确说明证据来源
- 内容正确：能覆盖搜索结果里的关键事实
- 反馈质量：不是只把搜索页原样吐回去，而是尽量整理成用户可直接消费的结论/步骤/提醒

评分口径：
- `PASS`：路由、证据、关键内容、结构服从均满足
- `WARN`：已完成搜索和证据反馈，但结构整理或可执行性偏弱
- `FAIL`：空回复、路由错误、无证据来源、明显漏关键信息

5 轮覆盖：
1. 公司评价：`xxx 公司怎么样`
2. 医院流程：`xxx 产检怎么预约`
3. 政务预约：`xxx 业务怎么预约`
4. 产品调研：`xxx 产品怎么样`
5. 证件办理：`xxx 办理怎么预约`

本轮具体案例：
1. `FRS-001` 星海智研这家公司怎么样，整理成整体印象 / 可能优势 / 风险
2. `FRS-002` 安和妇产医院产检怎么预约，整理成预约方式 / 所需准备 / 注意事项
3. `FRS-003` 深圳居住证续签怎么预约，整理成步骤清单
4. `FRS-004` 云杉 CRM 怎么样，整理成亮点 / 吐槽 / 适用场景
5. `FRS-005` 城西政务中心护照办理怎么预约，整理成预约入口 / 办理材料 / 现场提醒
