- 模型名不存在
- 上游限流
- 上游超时
- 请求格式不兼容
- 流式响应解析失败

## 11. 区分应用 500 还是网关 500

查看响应头里的 `server`：

```powershell
try {
    Invoke-WebRequest -Uri $Url -Method POST -Body $Body -ContentType "application/json" -UseBasicParsing -ErrorAction Stop
} catch {
    $_.Exception.Response.Headers
}
```

判断方向：

| 现象 | 可能原因 |
|---|---|
| `server: nginx` 返回 500 | 网关或后端应用异常 |
| `server: cloudflare` 且有 `cf-ray` | CDN / 边缘层或源站错误 |
| IIS 返回 500.x | IIS / ASP.NET / Windows 权限问题 |
| 响应很快 500 | 参数校验、配置、代码异常 |
| 等很久才 500 | 上游超时、数据库慢、模型接口超时 |
| 只有大消息 500 | token 长度、请求体限制、序列化失败 |
| 只有 stream=true 500 | SSE / 流式实现异常 |

```text
1. 用最小 body 复现 500
2. 记录 status、headers、request id
3. 查服务端应用日志
4. 查 IIS / nginx / 网关日志
5. 查 Windows 事件日志
6. 检查环境变量和密钥
7. 检查数据库、Redis、模型服务连通性
8. 对比正常请求和异常请求的 body 差异
```
