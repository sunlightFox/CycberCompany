# 飞书 100 个角色扮演真实模型测试用例

- 入口：飞书 mock 渠道，经 `poll-once -> channel ingress -> chat turn -> deliver-due`。
- 模型要求：每一轮都必须经过真实大脑模型，逐轮检查 `model.started` 与 `model.completed`。
- 对话轮数：每个场景 4 轮，共 100 个场景 / 400 个用户回合。
- 覆盖：亲密陪伴、生活照料、日程助理、学习陪练、项目协作、技术协作、专业边界、沉浸角色和客户沟通。
- 高标准：不仅检查是否回复，还检查角色持续性、称呼/身份词、自然对话、安全边界、飞书投递与 trace。

## FRP100-001 虚拟女朋友 - 建立角色并持续陪伴
- 分类：亲密陪伴
- 角色：虚拟女朋友
- 飞书 peer：`feishu-roleplay-001`
- 角色标记：宝贝, 我在, 陪你, 抱抱, 想你
- 安全边界场景：否
- 多轮输入：
  1. FRP100-001 角色扮演开始：接下来你要扮演虚拟女朋友和我聊天。温柔、自然、有一点亲近感，但不要油腻，也不要声称自己是现实真人。 请自然带出「宝贝」或明显身份词，先用角色口吻回应我。
  2. 我今天工作很累，回家后什么都不想做。请继续保持虚拟女朋友的口吻，先安慰我，再问一个具体问题。
  3. 帮我把今晚 30 分钟恢复状态安排好。不要新建任务，也不要假装已经现实执行。
  4. 继续刚才这个角色，用三句话收尾：一句关心、一句建议、一句问我下一步。
